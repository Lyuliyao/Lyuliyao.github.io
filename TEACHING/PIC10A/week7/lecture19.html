Part 1

Structs and Member Variables

The Problem: Loose Variables

            
int hour   = 9;
int minute = 30;
int second = 0;

// To pass "a time" around, every function needs three parameters:
void print_time(int h, int m, int s);
print_time(minute, hour, second);   // compiles, but wrong order!
  • Three variables that always travel together.
  • Nothing stops you from swapping the order at a call.
  • We want one value that is a time.

Declaring a struct

            
struct Time {
    int hour;
    int minute;
    int second;
};   // <-- semicolon is required
  • Time is now a type, usable like int.
  • hour, minute, second are member variables.
  • The closing brace must be followed by a semicolon.

Definition

A struct is a user-defined type that groups several member variables under one name. A variable of that type is called an instance; each instance has its own copy of every member.

Creating Instances

            
Time t1;        // an instance; its three int members are uninitialized
Time t2;        // a second, independent instance

// t1 and t2 are separate: 6 ints total, 3 per instance
  • Time t1; declares an instance the same way int n; does.
  • Each instance bundles all three members in one variable.
  • Members of built-in type start uninitialized here.

Initializing by Assignment

            
Time t2;
t2.hour   = 14;
t2.minute = 5;
t2.second = 45;
  • instance.member names one member of that instance.
  • Each member is assigned like an ordinary int.
  • This works, but takes one statement per member.

Initializing with Braces

            
Time t1{9, 30, 0};      // hour=9, minute=30, second=0
Time noon{12, 0, 0};    // members filled in declared order
  • Brace init fills members in the order they were declared.
  • {9, 30, 0} → hour=9, minute=30, second=0.
  • One concise line creates a fully set-up instance.

The Dot Operator

            
Time t{9, 30, 0};

std::cout << t.hour << ":" << t.minute << "\n";   // read: 9:30
t.minute = t.minute + 15;                          // write
std::cout << t.minute << "\n";                      // 45
  • t.hour reaches into one member of instance t.
  • Members are read and written like normal variables.
  • Writing t.minute changes only t's copy.

Each Instance Is Independent

            
Time a{1, 0, 0};
Time b{0, 1, 30};

a.hour = 5;                  // changes a only
std::cout << a.hour << "\n"; // 5
std::cout << b.hour << "\n"; // 0  (unaffected)
  • Two instances of the same type hold separate members.
  • Modifying a never touches b.
  • This is just like two separate int variables.

Passing a struct by Value

            
void show(Time t) {                      // a copy of the instance
    std::cout << t.hour << ":" << t.minute
              << ":" << t.second << "\n";
}

Time now{12, 0, 0};
show(now);   // ONE argument carries the whole time
  • One parameter replaces three loose ints.
  • By value copies all members into the parameter.
  • Changes inside show would not affect now.
Part 2

Member Functions

So far a struct only holds data. Now we put the behavior inside the type.

Behavior Outside the Type

            
struct Time { int hour; int minute; int second; };

// logic about a Time, but living far away from it:
int total_seconds(const Time& t) {
    return t.hour * 3600 + t.minute * 60 + t.second;
}

Time a{1, 0, 0};
std::cout << total_seconds(a) << "\n";   // 3600
  • The function and the data are separate.
  • Every access needs t. and an argument.
  • We can move this logic into the type.

Adding a Member Function

            
struct Time {
    int hour;
    int minute;
    int second;

    int total_seconds() {
        return hour * 3600 + minute * 60 + second;
    }
};
  • A member function is defined inside the struct.
  • It uses hour, minute, second with no t. prefix.
  • It takes no Time parameter — the instance is implicit.

Definition

A member function is a function declared inside a type. It is always called on an instance using the dot operator, and inside its body it can use that instance's member variables by name.

Calling on an Instance

            
Time a{1, 0, 0};   // 3600
Time b{0, 1, 30};  //   90

std::cout << a.total_seconds() << "\n";   // 3600
std::cout << b.total_seconds() << "\n";   //   90
  • instance.function() mirrors instance.member.
  • Same code, different instance ⇒ different result.
  • No Time argument: the instance is passed implicitly.

The Implicit Instance

In a.total_seconds(), the instance a is the implicit instance. An unqualified hour inside the body means that instance's hour — a.hour for this call, b.hour for the next.

Member Functions That Read

            
struct Time {
    int hour, minute, second;

    bool is_midnight() {                 // only reads members
        return hour == 0 && minute == 0 && second == 0;
    }
};

Time t{0, 0, 0};
std::cout << t.is_midnight() << "\n";   // 1 (true)
  • A reading member function only inspects members.
  • It leaves every member of the instance unchanged.
  • It can return a computed answer about the instance.

Member Functions That Modify

            
struct Time {
    int hour, minute, second;

    void add_minutes(int m) {            // changes members
        int total = total_seconds() + m * 60;
        hour   = (total / 3600) % 24;
        minute = (total / 60) % 60;
        second =  total % 60;
    }
    int total_seconds() {
        return hour * 3600 + minute * 60 + second;
    }
};
  • A modifying member function assigns to members.
  • It writes the implicit instance's own members.
  • It may call sibling member functions directly.

Modifying in Action

            
Time t{9, 30, 0};
t.add_minutes(45);

std::cout << t.hour   << "\n";   // 10
std::cout << t.minute << "\n";   // 15
std::cout << t.second << "\n";   //  0
  • t.add_minutes(45) rewrites t's members.
  • The change persists after the call returns.
  • Logic about a time now lives with the time.

Member Function vs Free Function

            
// Free function: data passed in explicitly
int total_seconds(const Time& t) {
    return t.hour * 3600 + t.minute * 60 + t.second;
}
int s1 = total_seconds(a);

// Member function: called ON the instance
int s2 = a.total_seconds();
  • Both compute the same thing.
  • The free function takes the instance as a parameter.
  • The member function lives inside the type and is called with the dot.
  • Keeping behavior next to its data makes the type easier to use.
Part 3

const Instances and const Member Functions

Some instances should never change after creation. We use const to make that promise — and the compiler enforces it.

A const Instance

            
struct Time {
    int hour, minute;

    int getHour() { return hour; }
    void setHour(int h) { hour = h; }
};

const Time noon{12, 0};   // a const instance
noon.setHour(13);         // ERROR: would modify noon
  • const Time noon may never change after creation.
  • Brace init still runs — that is initialization, not modification.
  • setHour would change it, so the compiler forbids the call.

The Surprise: Even Getters Are Rejected

            
const Time noon{12, 0};
int h = noon.getHour();   // ERROR, even though it only reads
  • getHour only reads, but the compiler does not know that.
  • From its declaration, getHour could modify the instance.
  • On a const instance, only functions that promise not to modify are allowed.

The Rule

A const instance may call only member functions that are themselves marked const. A member function makes that promise by writing const after its parameter list.

Marking a Member Function const

            
struct Time {
    int hour, minute;

    int getHour() const { return hour; }     // trailing const
    int getMinute() const { return minute; }
    void setHour(int h) { hour = h; }        // not const
};

const Time noon{12, 0};
std::cout << noon.getHour();   // OK now: getHour is const
  • The trailing const applies to the implicit instance.
  • It promises: "this function will not modify the instance."
  • const instances may call getHour and getMinute.

Where the const Goes

            
struct Time {
    int hour, minute;

    int  getHour() const { return hour; }   // reader: const
    void setHour(int h)  { hour = h;     }   // writer: no const
};
  • The const sits after the parameter list, before the body.
  • It is part of the function's signature, not a return-type detail.
  • Mark every member function that only reads as const; leave writers non-const.

What a const Member Function May Not Do

            
struct Time {
    int hour, minute;

    void show() const {
        hour = 0;          // ERROR: modifies a member
        setHour(9);        // ERROR: calls a non-const member
        std::cout << hour << ":" << minute;   // reading is fine
    }
    void setHour(int h) { hour = h; }
};
  • It must not modify any member variable.
  • It must not call a non-const member function.
  • It may freely read members and print them.

Inside a const Member, the Instance Is const

Within a const member function the implicit instance is treated as const. Any attempt to modify a member, or to call a non-const member function, is a compile error — the promise is enforced, not just documented.

Passing Instances by const Reference

            
void show(const Time& t) {
    std::cout << t.getHour();   // OK: getHour is const
    t.setHour(10);              // ERROR: setHour is not const
}

Time now{12, 0};
show(now);          // no copy, cannot modify now
  • const Time& avoids a copy and forbids changes.
  • Inside, t behaves like a const instance.
  • Only const member functions may be called on it.

The Classic Compile Error

            
struct Time {
    int hour, minute;
    int getHour() { return hour; }   // NOT const
};

void report(const Time& t) {
    std::cout << t.getHour();   // error here
}
  • "passing 'const Time' as 'this' argument discards qualifiers"
  • t is const, but getHour does not promise const.
  • Fix: mark getHour() const. Never remove const from the parameter.

Lecture Summary

Main ideas

  • struct bundles related member variables into one type.
  • Member functions put behavior inside the type; called with the dot.
  • Bare member names mean the implicit instance's members.
  • A const instance may call only const member functions.
  • Trailing const promises the function will not modify the instance.

Programming habits

  • Never forget the semicolon after the closing brace of a struct.
  • Keep behavior next to the data it uses.
  • Mark every read-only member function const.
  • Pass big instances by const T& to avoid copies and protect them.

Review Questions

Core ideas

  • Q1. How do you reach one member of an instance?
  • Answer: The dot operator: instance.member.
  • Q2. What is the implicit instance inside a member function?
  • Answer: The instance before the dot; its members are used by bare name.
  • Q3. Which member functions may a const instance call?
  • Answer: Only those marked const.

Reasoning

  • Q4. Why pass const Time& instead of Time?
  • Answer: Avoids copying the instance and guarantees it is not modified.
  • Q5. Where does the keyword const go on a member function?
  • Answer: After the parameter list, before the body.
  • Q6. May a const member function modify a member?
  • Answer: No; it may only read members and call other const members.

Code Review

What prints?

  • Q1. Output? 
                
struct Counter {
    int n;

};
Counter a{0}, b{0};
a.bump(); a.bump();
b.bump();
std::cout << a.n << " " << b.n << "\n";
  • Answer: 2 1 — each instance has its own n; bump() modifies the implicit instance only.

Spot the bug

  • Q2. What is the error, and the fix? 
                
struct Time {
    int hour, minute;
    int getMinute() { return minute; }
};

void show(const Time& t) {
    std::cout << t.getMinute();
}
  • Answer: t is const but getMinute is not; the call discards qualifiers. Fix: declare int getMinute() const.
