Constructors I

A constructor is a special member function that runs the moment an object is created. Its one job: leave the object in a valid, fully initialized state.

The Problem: Uninitialized Objects

            
struct Time {
    int hour;
    int minute;
};

Time t;             // hour, minute are GARBAGE
cout << t.hour;     // undefined behavior
t.hour = 25;        // nothing stops an invalid time
  • Nothing forces t into a sensible state.
  • Reading t.hour before setting it is undefined behavior.
  • We want every Time to start out valid.

Hand-Initialization Does Not Scale

            
Time t;
t.hour = 9;         // every caller must remember
t.minute = 30;      // every field, every time

Time other;         // forgot to set minute -> bug
cout << other.minute;   // garbage
  • Setup is repeated at every creation site.
  • One forgotten line is a silent, hard-to-find defect.
  • The type should initialize itself.

What a Constructor Is

A constructor is a member function with the same name as the type and no return type. It runs automatically when an object is created and is responsible for initializing the members.

A Default Constructor

            
struct Time {
    int hour;
    int minute;

    Time() {            // takes no arguments
        hour = 0;
        minute = 0;
    }
};

Time t;             // calls Time(): t is 00:00
  • No return type; name matches the type exactly.
  • A constructor that takes no arguments is the default constructor.
  • Time t; now produces a valid object, automatically.

When the Constructor Runs

            
struct Time {
    int hour, minute;

    Time() {
        cout << "Time built" << endl;
        hour = 0; minute = 0;
    }
};

Time a;             // prints "Time built"
Time b;             // prints "Time built" again
  • You never call the constructor by name.
  • It fires once per object, at the point of creation.
  • Two objects ⇒ two constructor runs.

Parameterized Constructors

            
struct Time {
    int hour;
    int minute;

    Time() { hour = 0; minute = 0; }

    Time(int h, int m) {        // parameterized
        hour = h;
        minute = m;
    }
};

Time start;            // 00:00  -> default
Time lunch(12, 30);    // 12:30  -> parameterized
  • Constructors can be overloaded like any function.
  • The compiler picks one by the construction arguments.
  • One type, several ways to build it.

A Constructor's Job: A Valid Initial State

A constructor should leave the object in a state where every member function can be called safely. After Time lunch(12, 30);, both hour and minute hold meaningful values — never garbage.

Constructing Objects Several Ways

            
Time a;             // default ctor    -> 00:00
Time b(9, 15);      // (int,int) ctor  -> 09:15
Time c{9, 15};      // brace form      -> 09:15
Time d = Time(9,15);// build temp, then copy

Time e();           // NOT an object: declares a function
  • Each line selects a constructor by its arguments.
  • Time e(); is the classic "most vexing parse" trap.
  • Use Time e; (no parentheses) for the default ctor.

Constructors and Containers

            
struct Time {
    int hour, minute;

    Time() { hour = 0; minute = 0; }
    Time(int h, int m) { hour = h; minute = m; }
};

vector<Time> day(3);          // 3 elements, each Time()
day[0] = Time(8, 0);          // overwrite element 0
  • vector<Time> day(3); calls the default constructor once per slot.
  • So a type stored in a container usually needs a default constructor.
  • Each element is valid the instant the vector exists.

A Running Example: BankAccount

            
struct BankAccount {
    double balance;

    BankAccount() {
        balance = 0.0;
    }
    BankAccount(double initial) {
        balance = initial;
    }
    void deposit(double amt) { balance += amt; }
    double get() { return balance; }
};

BankAccount empty;             // 0.00
BankAccount savings(500.0);    // 500.00
savings.deposit(50.0);         // 550.00
  • Every account starts with a defined balance.
  • No caller can forget to initialize balance.
  • Construction guarantees a valid balance from line one.

The Constructor Sees the Members Directly

            
struct BankAccount {
    double balance;

    BankAccount(double initial) {
        if (initial < 0.0)
            balance = 0.0;      // refuse a bad start
        else
            balance = initial;
    }
};

BankAccount a(-40.0);   // balance becomes 0.00, not -40
  • The body is ordinary code that runs at creation.
  • It can reject bad input and still produce a valid object.
  • That is the whole point: a guaranteed sane starting state.

Lecture 21 Summary

Main ideas

  • A constructor has the type's name and no return type.
  • It runs automatically when an object is created.
  • The default constructor takes no arguments.
  • Constructors can be overloaded; arguments pick one.

Programming habits

  • Initialize every member; never leave garbage.
  • A constructor must leave a valid initial state.
  • Provide a default constructor for types kept in containers.

Lecture 21 Review Questions

Core ideas

  • Q1. What return type does a constructor have?
  • Answer: None at all, not even void.
  • Q2. What is the default constructor?
  • Answer: The constructor that can be called with no arguments.
  • Q3. When does a constructor run?
  • Answer: Automatically, once, when the object is created.

Reasoning

  • Q4. Which constructor does Time lunch(12, 30); call?
  • Answer: The parameterized Time(int, int), chosen by overload resolution.
  • Q5. Why does vector<Time> v(3); need a default constructor?
  • Answer: It builds three elements, calling Time() once for each.
  • Q6. What is a constructor's responsibility?
  • Answer: To leave the object in a valid, fully initialized state.

Lecture 21 Code Review

Find the bug

  • Q1. What is wrong here? 
                
struct Time {
    int hour, minute;

    void Time() {           // ??
        hour = 0; minute = 0;
    }
};
  • Answer: A constructor has no return type. Writing void makes it an ordinary member function named Time, not a constructor; remove void.

Trace the value

  • Q2. What is the final balance? 
                
struct BankAccount {
    double balance;

    BankAccount() { balance = 0.0; }
    BankAccount(double b) { balance = b; }
    void deposit(double a) { balance += a; }
    double get() { return balance; }
};

BankAccount acct(100.0);
acct.deposit(25.0);
cout << acct.get();
  • Answer: 125. The (double) constructor sets 100.0, then deposit adds 25.0.
