Pointers I

A pointer is a variable whose value is a memory address. Today we learn how to take an address, store it, and use it to read or write the variable it points at.

addresses & and * nullptr pointers vs references

Where We Are

L22 left us with

  • Classes that own and manage their data.
  • References for passing without copying.
  • Default arguments to simplify call sites.

Today

  • Every variable lives at an address in memory.
  • A pointer stores such an address.
  • Dereferencing reaches the variable behind the address.

Memory Is Numbered

Think of memory as a long row of numbered boxes. Each variable occupies one or more boxes. The address of a variable is the number of its first box.

The Address-of Operator &

            
int x = 42;
std::cout << x;     // 42       -- the value
std::cout << &x;    // 0x7ffd.. -- the address of x
  • x is the value stored in the box.
  • &x is "the address of x" — where the box is.
  • The exact number varies per run; only the idea matters.

Declaring a Pointer

            
int  x = 42;
int* p = &x;       // p holds the address of x

// read it back:
std::cout << p;     // the address (same as &x)
std::cout << &x;    // the same address
  • int* means "pointer to an int."
  • p = &x stores the address of x in p.
  • The type of the pointee matters: int* ≠ double*.

A Picture in Text

            
int  x = 42;
int* p = &x;

//  name   addr        value
//  x      1000        42
//  p      1004        1000   (points at x)
  • x lives at address 1000 and stores 42.
  • p lives at 1004 and stores 1000.
  • "p points at x" = p's value is x's address.

The Dereference Operator *

            
int  x = 42;
int* p = &x;

std::cout << *p;   // 42  -- follow p to x, read x
*p = 99;           // follow p to x, write x
std::cout << x;    // 99  -- x changed through p
  • *p means "the variable p points to."
  • *p on the right reads; on the left it writes.
  • Writing *p changes x itself.

Two Meanings of *

In a declaration, int* p means "p is a pointer." In an expression, *p means "dereference p." Same symbol, different jobs.

Null Pointers

            
int* p = nullptr;       // points at nothing

if (p != nullptr)
    std::cout << *p;     // safe: only deref when valid

std::cout << *p;         // CRASH: dereferencing nullptr
  • nullptr is the "points at nothing" value.
  • Dereferencing a null pointer is undefined behavior (a crash).
  • Check p != nullptr before *p.

Pointing at a Local Variable

            
int main() {
    int score = 88;
    int* p = &score;

    *p += 10;                 // modify score via p
    std::cout << score;       // 98
    std::cout << *p;          // 98
}
  • p aliases score: two names, one box.
  • *p and score always agree while p points there.
  • Valid only while score is alive (more next lecture).

Pointers vs References

            
int x = 1, y = 2;

int& r = x;     // reference: bound to x forever
r = y;          // x becomes 2 (NOT a rebind)

int* p = &x;    // pointer: can be reseated
p = &y;         // now p points at y instead
*p = 5;         // y becomes 5
  • A reference binds once and is never null.
  • A pointer can be reseated and can be nullptr.
  • A reference needs no *; a pointer needs * to reach the value.

Lecture 23 Summary

Main ideas

  • A pointer holds an address; int* p declares one.
  • &x takes the address of x.
  • *p reaches the variable p points to.
  • nullptr means "points at nothing."

Habits

  • Initialize pointers (to an address or nullptr).
  • Never dereference a pointer you have not checked.
  • Prefer a reference when you never need null or reseating.

Lecture 23 Review Questions

Core ideas

  • Q1. What does a pointer variable store?
  • Answer: A memory address (the location of another variable).
  • Q2. What does &x evaluate to?
  • Answer: The address of x.
  • Q3. What does *p mean in an expression?
  • Answer: The variable that p points to.

Reasoning

  • Q4. After int x=5; int* p=&x; *p=8;, what is x?
  • Answer: 8 — writing through p changes x.
  • Q5. Name one thing a pointer can do that a reference cannot.
  • Answer: Be null, or be reseated to point elsewhere.
  • Q6. Why crash on *p when p == nullptr?
  • Answer: There is no variable at the null address to read or write.

Lecture 23 Code Review

Trace the output

  • Q1. What prints? 
                
int a = 10;
int b = 20;
int* p = &a;
*p = 15;
p = &b;
*p = 25;
std::cout << a << ' ' << b;
  • Answer: 15 25. *p=15 changes a; then p is reseated to b, and *p=25 changes b.

Spot the bug

  • Q2. What is wrong here? 
                
int* p;            // uninitialized
std::cout << *p;   // ?
  • Answer: p holds a garbage address; dereferencing it is undefined behavior. Initialize: int* p = nullptr; or to a real address.
