Part I Roadmap

Indexing s[0] means first character Strings use zero-based indexing.
Safety at() checks bounds [] does not.

Zero-Based Indexing

            
string alphabet = "abcdefghijklmnopqrstuvwxyz";

cout << alphabet[0] << endl;
cout << alphabet[1] << endl;
cout << alphabet[25] << endl;
cout << alphabet[alphabet.length() - 1] << endl;
  • Character indexing starts at 0.
  • The last character is at index length() - 1.
  • If a string has length n, its character indices run from 0 to n - 1.

[] Versus at()

[]

  • Short and common.
  • No bounds checking.
  • s[pos] is undefined behavior when pos > s.length().
  • s[s.length()] is usually a mistake, even though the textbook notes it is not undefined behavior.

at()

  • Member function call.
  • Checks bounds.
  • s.at(pos) gives a runtime error when pos >= s.length().

string s = "abc" looks like [0]='a' | [1]='b' | [2]='c' | [3]='\0', so s[s.length()] touches the reserved terminator slot.

Mutating Characters

            
string s = "bruin";

s[0] = 'B';
s.at(1) = 'R';
s.at(2) += 'A' - 'a';

cout << s << endl;
  • Indexing is not only for reading. It can also change characters.
  • The same caution about bounds still applies.
  • Character arithmetic still works inside strings because each element is a char.

Out-of-Bounds Rules

Undefined behavior

  • s[pos] with pos > s.length()
  • s[s.length()] = ch when ch != '\0'

Runtime error

  • s.at(pos) with pos >= s.length()
While learning, at() is often better because it fails loudly instead of silently corrupting your assumptions.

Why Does size_t Keep Appearing?

            
#include <cstddef>
#include <string>

string s = "hello";
size_t n = s.length();

cout << n << endl;
  • length() and size() return std::size_t.
  • It is an unsigned integer type meant for sizes and indices.
  • We will keep seeing size_t in indexing and searching.

substr

            
string campus = "Westwood";

cout << campus.substr(0, 4) << endl;
cout << campus.substr(4, 4) << endl;
cout << campus.substr(2, 3) << endl;
  • The first argument is the starting index.
  • The second argument is how many characters to take.
  • The result is a new string; campus itself is unchanged.

substr Edge Cases

            
string s = "0123456789";

cout << s.substr(4) << endl;
cout << s.substr(4, 44) << endl;
cout << "[" << s.substr(10) << "]" << endl;
// cout << s.substr(11) << endl;
  • substr(start) returns everything from start to the end.
  • If count is too large, it just uses the remaining characters.
  • If start == s.length(), the result is the empty string.
  • If start > s.length(), a runtime error follows.

push_back and pop_back

            
string s = "Bruin";

s.push_back('s');
cout << s << endl;

s.pop_back();
cout << s << endl;
  • push_back(ch) adds one character at the end.
  • pop_back() removes the last character.
  • These are convenient when building or trimming text step by step.

find and rfind

            
string s = "bananas";

size_t p1 = s.find("ana");
size_t p2 = s.rfind("ana");

cout << p1 << endl;
cout << p2 << endl;
  • find looks from the left.
  • rfind looks from the right.
  • Both return positions, so the type is size_t.

When Search Fails: npos

            
string school = "ucla";

size_t pos = school.find("x");
cout << (pos == string::npos) << endl;
  • If the target is not found, the answer is string::npos.
  • Do not guess a fake index like -1.
  • Because the result type is size_t, comparing to string::npos is the standard pattern.

A Common find Bug

            
string name = "amy";

if (name.find("a")) {
    cout << "found" << endl;
}
  • This code is wrong when the answer is position 0.
  • 0 converts to false, so the branch is skipped.
  • Use name.find("a") != string::npos instead.

Lecture 5 Review Questions

Indexing and Bounds

  • Q1. If string s = "cat";, what does s[0] refer to?
  • Answer: The first character, 'c'.
  • Q2. If a string has length n, what is the last valid index?
  • Answer: n - 1
  • Q3. What is the main safety difference between [] and at()?
  • Answer: at() checks bounds, while [] does not.
  • Q4. Why is s[s.length()] suspicious?
  • Answer: Because it touches the reserved '\0' slot instead of an ordinary character.

String Tools

  • Q5. What does substr(2, 3) mean?
  • Answer: Start at index 2 and take 3 characters.
  • Q6. What does a failed search return?
  • Answer: string::npos
  • Q7. Why is if (s.find("a")) dangerous?
  • Answer: Because a match at index 0 converts to false.
  • Q8. What type do length(), find, and rfind return?
  • Answer: size_t

Lecture 5 Code Review

Undefined behavior or not?

  • Q1. Classify the issue. 
                      
string s = "0123";
cout << s[s.length()] << endl;
  • Answer: Not undefined behavior, but usually a mistake. It reads the reserved '\0' slot, so nothing visible is printed before endl.

Spot the logic bug

  • Q2. Why can this fail even when the string contains "a"? 
                      
string name = "amy";
if (name.find("a")) {
    cout << "found" << endl;
}
  • Answer: The match is at index 0, and 0 converts to false. The test should compare against string::npos.

Lecture 5 Summary

Main ideas

  • Strings use zero-based indexing.
  • [] is unchecked; at() is checked.
  • substr extracts smaller strings.
  • push_back and pop_back grow and shrink strings.
  • find and rfind return positions.

Programming habits

  • Watch bounds carefully whenever length() appears in an index expression.
  • Use at() while learning or debugging.
  • Read substr(start, count) as start first, then how many.
  • Test searches with != string::npos.
