Member Functions

A struct can do more than hold data. We put the functions that act on a value inside the type, so behavior lives next to the data it uses.

Behavior Outside the Type

            
struct Time { int hour; int minute; int second; };

// logic about a Time, but living far away from it:
int total_seconds(const Time& t) {
    return t.hour * 3600 + t.minute * 60 + t.second;
}

Time a{1, 0, 0};
std::cout << total_seconds(a) << "\n";   // 3600
  • The function and the data are separate.
  • Every access needs t. and an argument.
  • We can move this logic into the type.

Adding a Member Function

            
struct Time {
    int hour;
    int minute;
    int second;

    int total_seconds() {
        return hour * 3600 + minute * 60 + second;
    }
};
  • A member function is defined inside the struct.
  • It uses hour, minute, second with no t. prefix.
  • It takes no Time parameter — the object is implicit.

Definition

A member function is a function declared inside a type. It is always called on an object using the dot operator, and inside its body it can use that object's member variables by name.

Calling on an Object

            
Time a{1, 0, 0};   // 3600
Time b{0, 1, 30};  //   90

std::cout << a.total_seconds() << "\n";   // 3600
std::cout << b.total_seconds() << "\n";   //   90
  • object.function() mirrors object.member.
  • Same code, different object ⇒ different result.
  • No Time argument: the object is passed implicitly.

The Implicit Object

In a.total_seconds(), the object a is the implicit object. An unqualified hour inside the body means that object's hour — a.hour for this call, b.hour for the next.

Which Object's Members?

            
struct Time {
    int hour, minute, second;
    int minutes_past() { return hour * 60 + minute; }
};

Time morning{9, 30, 0};
Time evening{18, 45, 0};

std::cout << morning.minutes_past() << "\n"; // 570  (9*60+30)
std::cout << evening.minutes_past() << "\n"; // 1125 (18*60+45)
  • hour resolves to the object before the dot.
  • One function body serves every Time object.
  • The implicit object selects which members are read.

Member Functions That Read

            
struct Time {
    int hour, minute, second;

    bool is_midnight() {                 // only reads members
        return hour == 0 && minute == 0 && second == 0;
    }
};

Time t{0, 0, 0};
std::cout << t.is_midnight() << "\n";   // 1 (true)
  • A reading member function only inspects members.
  • It leaves every member of the object unchanged.
  • It can return a computed answer about the object.

Member Functions That Modify

            
struct Time {
    int hour, minute, second;

    void add_minutes(int m) {            // changes members
        int total = total_seconds() + m * 60;
        hour   = (total / 3600) % 24;
        minute = (total / 60) % 60;
        second =  total % 60;
    }
    int total_seconds() {
        return hour * 3600 + minute * 60 + second;
    }
};
  • A modifying member function assigns to members.
  • It writes the implicit object's own members.
  • It may call sibling member functions directly.

Modifying in Action

            
Time t{9, 30, 0};
t.add_minutes(45);

std::cout << t.hour   << "\n";   // 10
std::cout << t.minute << "\n";   // 15
std::cout << t.second << "\n";   //  0
  • t.add_minutes(45) rewrites t's members.
  • The change persists after the call returns.
  • Logic about a time now lives with the time.

Building on Point

            
struct Point {
    double x;
    double y;

    double distance_to(const Point& p) {     // reads
        double dx = x - p.x;
        double dy = y - p.y;
        return std::sqrt(dx * dx + dy * dy);
    }
    void translate(double dx, double dy) {   // modifies
        x = x + dx;
        y = y + dy;
    }
};

Point origin{0, 0};
Point q{3, 4};
std::cout << origin.distance_to(q) << "\n"; // 5
q.translate(1, 1);                           // q is now (4, 5)
  • x, y are the implicit object's; p.x, p.y are the argument's.
  • One type carries both data and the behavior on it.

Member Function vs Free Function

            
// Free function: data passed in explicitly
int total_seconds(const Time& t) {
    return t.hour * 3600 + t.minute * 60 + t.second;
}
int s1 = total_seconds(a);

// Member function: called ON the object
int s2 = a.total_seconds();
  • Both compute the same thing.
  • The free function takes the object as a parameter.
  • The member function lives inside the type and is called with the dot.
  • Keeping behavior next to its data makes the type easier to use.

Lecture 20 Summary

Main ideas

  • Member functions put behavior inside the type.
  • They are called on an object with the dot operator.
  • Bare member names mean the implicit object's members.
  • A member function is just a free function moved inside the type.

Programming habits

  • Keep behavior next to the data it uses.
  • Let modifying functions write the implicit object.
  • Reuse sibling member functions instead of repeating logic.

Lecture 20 Review Questions

Core ideas

  • Q1. How do you call a member function?
  • Answer: On an object with the dot: object.function().
  • Q2. What is the implicit object?
  • Answer: The object before the dot; its members are used by bare name.
  • Q3. In a.total_seconds(), whose hour is used?
  • Answer: a's — a is the implicit object.

Reasoning

  • Q4. Can a member function change the object's members?
  • Answer: Yes; it can assign to the implicit object's members, e.g. add_minutes.
  • Q5. What does a member function gain over a free function that takes the object?
  • Answer: It is called with the dot and refers to the object's members by bare name — behavior stays next to its data.
  • Q6. In distance_to(const Point& p), what is p.x vs bare x?
  • Answer: p.x is the argument's; bare x is the implicit object's.

What prints?

  • Q1. Output? 
                
struct Counter {
    int n;
    void bump() { n = n + 1; }
};
Counter a{0}, b{0};
a.bump(); a.bump();
b.bump();
std::cout << a.n << " " << b.n << "\n";
  • Answer: 2 1 — each object has its own n; bump() modifies the implicit object only.

Spot the bug

  • Q2. What does this print? 
                
struct Time {
    int hour, minute, second;
    int total() { return hour * 3600 + minute * 60 + second; }
    void add_minutes(int m) { minute = minute + m; }
};

Time t{9, 30, 0};
t.add_minutes(15);
std::cout << t.total() << "\n";
  • Answer: 34500. add_minutes(15) writes the implicit object's minute (now 45), then total() computes 9×3600 + 45×60 + 0.
