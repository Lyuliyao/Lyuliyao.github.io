Functions & Overloading

            
int    area(int s)            { return s * s; }
double area(double w, double h) { return w * h; }

area(4);          // calls int overload  -> 16
area(2.0, 3.0);   // calls double overload -> 6.0
  • Overloads must differ in parameter types, not just return.
  • The compiler picks the best viable match.
  • Takeaway: same idea, same name; different data, different signature.

References & const

            
void grow(int& n)            { n *= 2; }       // modifies caller
int  sum(const vector<int>& v) {              // no copy, read-only
    int s = 0; for (int x : v) s += x; return s;
}
  • T& aliases the caller's object.
  • const T& avoids a copy and forbids changes.
  • Takeaway: pass big objects by const& by default.

Headers & Separate Compilation

            
// shape.h  — declarations only
#ifndef SHAPE_H
#define SHAPE_H
double area(double r);
#endif

// shape.cpp — one definition
#include "shape.h"
double area(double r) { return 3.14159 * r * r; }
  • Headers declare; .cpp files define.
  • Include guards stop double inclusion.
  • Takeaway: one definition rule — declare often, define once.

Structs, Classes & Constructors

            
class Point {
public:
    Point(double x, double y) : x_(x), y_(y) {}
    double getX() const { return x_; }
private:
    double x_, y_;
};

Point p(1.0, 2.0);   // constructor runs
  • A class bundles data with the functions on it.
  • Constructors initialize via the member initializer list.
  • Takeaway: an object is never left half-built.

const Members & Encapsulation

            
class Account {
public:
    double balance() const { return bal_; }   // promises no change
    void deposit(double d) { bal_ += d; }
private:
    double bal_ = 0.0;                          // hidden state
};
  • const member functions cannot modify the object.
  • private data is reached only through the interface.
  • Takeaway: invariants stay safe behind a small public API.

Pointers

            
int x = 7;
int* p = &x;     // p holds the address of x
*p = 9;           // x is now 9
cout << *p << endl;   // 9
  • & takes an address; * dereferences.
  • A pointer can be reseated; a reference cannot.
  • Takeaway: pointers give explicit, rebindable indirection.

C-style Arrays

            
int a[4] = {1, 2, 3, 4};
int* p = a;            // decay to &a[0]
a[2] == *(a + 2);      // true — both name 8... err, 3

void f(int a[], int n);   // size must travel separately
  • Fixed size, indices 0..N-1, no bounds checks.
  • Arrays decay to pointers; a[i] == *(a + i).
  • Takeaway: prefer std::vector unless forced otherwise.

Common Exam Pitfalls

Easy points lost

  • Overloads that differ only by return type.
  • Returning a reference to a local variable.
  • Forgetting const on a read-only member.
  • Missing include guards.

Memory traps

  • i <= N loops walking off the array.
  • sizeof on a decayed array parameter.
  • C-string with no '\0'.
  • Dereferencing the one-past-end pointer.

Final Review Questions

Concepts

  • Q1. Why pass a vector as const&?
  • Answer: Avoids a copy and guarantees the function won't modify it.
  • Q2. What does a const member function promise?
  • Answer: It will not modify the object's data members.
  • Q3. Where does the one definition rule put a function body?
  • Answer: In exactly one .cpp file; the header only declares it.

Memory

  • Q4. What is a[i] in pointer form?
  • Answer: *(a + i).
  • Q5. Why pass the size with a raw array?
  • Answer: It decays to a pointer and loses its length.
  • Q6. Difference between a pointer and a reference?
  • Answer: A pointer can be null and reseated; a reference is a fixed alias.

Final Code Review

Spot the bug

  • Q1. What is wrong? 
                
int& first(int a[], int n) {
    int x = a[0];
    return x;          // returning ref to a local
}
  • Answer: x dies when the function returns; the reference dangles. Return a[0] by value, or return a[0]; as int&.

What prints?

  • Q2. Output? 
                
class C {
public:
    C(int v) : v_(v) {}
    int get() const { return v_; }
private:
    int v_;
};
C c(5);
cout << c.get() << endl;
  • Answer: 5. The constructor sets v_; the const getter reads it.

A Taste of 10B

            
template <typename T>
T maxOf(const T& a, const T& b) {
    return (a < b) ? b : a;
}

maxOf(3, 7);          // 7
maxOf(2.5, 1.5);      // 2.5
maxOf(std::string("ant"), std::string("bee"));  // "bee"
  • One template serves every comparable type.
  • This is how the STL containers are built.
