Part A

Designing a Class End to End

Design Goal

Build a Gradebook that owns a std::vector<int> of scores. The class invariant: every stored score is in 0..100. We will pick members, design the interface, write constructors with init lists, and split into .hpp/.cpp.

Step 1: Design Decisions

Data members one vector A std::vector<int> holds the scores.
const? readers are const size, at, average never modify state.
Invariant scores in 0..100 The class rejects values outside the range.

Step 2: The Header (Interface)

            
// gradebook.hpp
#ifndef GRADEBOOK_HPP
#define GRADEBOOK_HPP
#include <vector>

class Gradebook {
public:
    Gradebook();                 // start empty
    void add(int score);         // append, enforce 0..100
    int  size() const;           // how many scores
    int  at(int i) const;        // read score i
    double average() const;      // mean of all scores
private:
    std::vector<int> scores;     // the managed data
};
#endif
  • scores is private: callers cannot touch it.
  • Three of four functions are const — readers only.
  • add is the one door for new data, so it enforces the rule.

Step 3: The Implementation (.cpp)

            
#include "gradebook.hpp"

Gradebook::Gradebook() : scores() { }   // empty vector

void Gradebook::add(int score) {
    if (score < 0)   score = 0;
    if (score > 100) score = 100;
    scores.push_back(score);            // invariant kept
}
int Gradebook::size() const { return scores.size(); }
int Gradebook::at(int i) const { return scores[i]; }
double Gradebook::average() const {
    if (scores.size() == 0) return 0.0;
    int sum = 0;
    for (int i = 0; i < scores.size(); ++i) sum += scores[i];
    return static_cast<double>(sum) / scores.size();
}
  • The constructor uses an init list (scores() = empty).
  • add clamps before push_back — invariant held.
  • const members read scores but never modify it.

Step 4: Trace It

            
Gradebook g;
g.add(80);
g.add(120);          // clamped to 100
g.add(-5);           // clamped to 0
std::cout << g.size()    << '\n';   // ?
std::cout << g.at(1)     << '\n';   // ?
std::cout << g.average() << '\n';   // ?
  • size() → 3 (three calls to add).
  • at(1) → 100 (the clamped 120).
  • average() → (80+100+0)/3 = 60.

Step 5: const Instances Need const Members

            
void report(const Gradebook& gb) {
    std::cout << gb.size();      // OK: size() is const
    std::cout << gb.average();   // OK: average() is const
    gb.add(50);                  // ERROR: add() is non-const
}
  • A const Gradebook& can only call const members.
  • This is why we marked the readers const.
  • The design choices from Step 1 pay off here.
Part B

Default Arguments

One function, two ways to call it — with or without trailing arguments.

What a Default Argument Is

A default argument supplies a value the caller may omit. One function then serves the common case and the customized case — Chapter 16.

Default Argument Syntax

            
void greet(std::string name, std::string punct = "!") {
    std::cout << "Hello, " << name << punct << "\n";
}

greet("Ada");          // Hello, Ada!
greet("Ada", ".");     // Hello, Ada.
greet("Ada", "?");     // Hello, Ada?
  • = "!" is the default for punct.
  • Omitting it uses the default.
  • Supplying it overrides the default.

Defaults Go in the Declaration

            
// log.hpp
void log(std::string msg, int level = 1);   // default HERE

// log.cpp
#include "log.hpp"
void log(std::string msg, int level) {      // NO default here
    std::cout << "[" << level << "] " << msg << "\n";
}
  • The default lives where callers see it — the header.
  • Repeating it in the definition is an error.
  • If there is no header, put it on the single declaration.

The Trailing-Parameter Rule

            
void box(int w, int h = 1, int d = 1);   // OK
void bad(int w = 1, int h);              // error: h has no default
  • Once a parameter has a default, all after it must too.
  • Defaults are filled from the right end leftward.
  • bad could never omit w while passing h.

Omitting Only Trailing Arguments

            
void box(int w, int h = 1, int d = 1);

box(5);          // w=5, h=1, d=1
box(5, 2);       // w=5, h=2, d=1
box(5, 2, 3);    // w=5, h=2, d=3
box(5, , 3);     // ERROR: cannot skip the middle one
  • You may only drop arguments from the right.
  • There is no syntax to skip a middle argument.
  • To set d you must also pass h.

The Ambiguity Trap

            
void f(int a);                 // candidate A
void f(int a, int b = 0);      // candidate B

f(3, 4);   // calls B, fine
f(3);      // AMBIGUOUS: A matches, B matches via default
  • f(3) fits f(int) exactly.
  • It also fits f(int, int=0) with the default.
  • Two equally good matches ⇒ build error.

Avoiding the Trap

            
// Pick ONE strategy for the arity choice.

// Strategy 1: defaults only
void f(int a, int b = 0);

// Strategy 2: overloads only
void g(int a);
void g(int a, int b);
  • Use defaults or overloads, not both for the same call.
  • Do not let an overload shadow a defaulted call.
  • Clear interfaces compile without surprises.

Lecture 22 Summary

Main ideas

  • Design: pick data members, then the operations they need.
  • State the invariant; defend it in the constructor and add.
  • Defaults go in the declaration only.
  • Every parameter after a defaulted one must be defaulted.

Programming habits

  • Make every reader const so const instances stay usable.
  • Prefer one defaulted function over near-duplicate overloads.
  • Do not mix an overload with a default that covers it.

Lecture 22 Review Questions

Classes review

  • Q1. Why make scores private?
  • Answer: So callers cannot bypass add and store an out-of-range value.
  • Q2. Why is size() marked const?
  • Answer: It only reads scores, and const instances must be able to call it.
  • Q3. Why does average() check size() == 0 first?
  • Answer: To avoid dividing by zero on an empty gradebook.

Default arguments

  • Q4. Where does a default argument go?
  • Answer: In the declaration (header), not repeated in the definition.
  • Q5. Is void p(int a = 1, int b); legal?
  • Answer: No — all parameters after a defaulted one must be defaulted.
  • Q6. Why is f(3) ambiguous with f(int) and f(int,int=0)?
  • Answer: Both are viable for one argument and neither is strictly better.

Lecture 22 Code Review

Spot the bug

  • Q1. Why is this loop wrong? 
                
double Gradebook::average() const {
    int sum = 0;
    for (int i = 1; i <= scores.size(); ++i)
        sum += scores[i];
    return (double)sum / scores.size();
}
  • Answer: It starts at 1 (skips scores[0]) and uses <= (reads scores[size()], out of range). Use i = 0; i < scores.size().

Does it compile?

  • Q2. Build result? 
                
void scale(int v);
void scale(int v, double k = 2.0);

int main() { scale(10); }
  • Answer: Build error: scale(10) matches scale(int) exactly and scale(int,double=2.0) via the default — ambiguous.
