Pointers II

Now we put pointers to work: passing them to functions, mixing them with const, reaching members through a pointer with ->, and recognizing the dangling-pointer bugs that crash programs.

pointers + functions const pointers the -> operator dangling pointers

Where We Are

L23 left us with

  • &x takes an address; *p dereferences.
  • nullptr means "points at nothing."

Today

  • Functions can modify a caller's variable via a pointer.
  • const can protect the pointee or the pointer.
  • Reach an instance's members through a pointer with ->.
  • Lifetime bugs: dangling pointers.

Passing a Pointer to a Function

            
void add_ten(int* p) {
    *p += 10;             // modifies the caller's variable
}

int main() {
    int score = 50;
    add_ten(&score);      // pass the address
    std::cout << score;   // 60
}
  • The function receives a copy of the address, not the value.
  • Through *p it reaches the caller's score.
  • The caller writes &score at the call site.

Pointer vs Reference Parameter

            
void by_ptr(int* p) { *p += 1; }
void by_ref(int& r) {  r += 1; }

int n = 0;
by_ptr(&n);   // explicit & at call, * inside
by_ref(n);    // clean call, no * inside
  • Both let the function modify the caller's variable.
  • Reference: cleaner syntax, never null.
  • Pointer: can be null, so callers can pass "no instance."

const With Pointers: Two Choices

const int* p → pointer to const: you cannot change what it points to. int* const p → const pointer: you cannot reseat it. Read right to left from the variable name.

Pointer to const

            
int x = 1, y = 2;
const int* p = &x;   // pointer to const int

std::cout << *p;      // OK: read through p
*p = 5;               // ERROR: cannot write the pointee
p = &y;               // OK: p itself may be reseated
  • Use for a function that reads but must not modify the pointee.
  • void print(const int* p) promises "I won't change it."
  • The pointer variable is still free to point elsewhere.

const Pointer

            
int x = 1, y = 2;
int* const p = &x;   // const pointer to int

*p = 9;               // OK: change the pointee
std::cout << x;       // 9
p = &y;               // ERROR: cannot reseat p
  • p is locked to x for its whole life.
  • You may still modify x through *p.
  • const int* const p locks both pointer and pointee.

Reaching Members: the -> Operator

            
struct Point { int x; int y; };

Point pt{3, 4};
Point* p = &pt;

std::cout << (*p).x;   // dereference, then .x
std::cout << p->y;     // same thing, cleaner

p->x = 10;             // writes pt.x through the pointer
  • A pointer can point at an instance of a class/struct type.
  • p->member is shorthand for (*p).member.
  • Use -> whenever you have a pointer to an instance.

Dangling Pointers

            
int* make() {
    int local = 7;
    return &local;     // BUG: local dies on return
}

int* p = make();
std::cout << *p;       // dangling: undefined behavior
  • local is destroyed when make returns.
  • p still holds its old address — a dangling pointer.
  • Never return the address of a local variable.

Out-of-Scope Locals

            
int* p = nullptr;
{
    int temp = 42;
    p = &temp;         // valid only inside this block
}
std::cout << *p;       // BUG: temp is gone here
  • temp dies at the closing brace of its block.
  • p outlives temp → dangles.
  • A pointer must not outlive what it points to.

Check Before You Dereference

            
// Point defined earlier: struct Point { int x; int y; };

void print_point(const Point* p) {
    if (p == nullptr) {                 // guard first
        std::cout << "no point\n";
        return;
    }
    std::cout << p->x << ", " << p->y << "\n";   // safe now
}
  • Check for nullptr before any *p or p->.
  • Dereferencing a null pointer is undefined behavior — usually a crash.
  • const Point* documents that the instance is only read.

Lecture 24 Summary

Main ideas

  • A pointer parameter lets a function modify the caller's variable.
  • const int* protects the pointee; int* const locks the pointer.
  • p->m is (*p).m — reach members through a pointer.
  • A pointer must not outlive its target.

Habits

  • Use const T* for read-only pointer parameters.
  • Never return or keep the address of a local.
  • Guard nullptr before every dereference.

Lecture 24 Review Questions

Core ideas

  • Q1. How does void f(int* p) change a caller's variable?
  • Answer: The caller passes &var; f writes through *p.
  • Q2. What does const int* p forbid?
  • Answer: Writing through p (*p = ...); reseating p is allowed.
  • Q3. What does int* const p forbid?
  • Answer: Reseating p; modifying *p is allowed.

Reasoning

  • Q4. Given Point* p, what is p->x equivalent to?
  • Answer: (*p).x — dereference, then access the member.
  • Q5. Why is a pointer to an out-of-scope local dangerous?
  • Answer: The variable is destroyed; dereferencing is undefined behavior.
  • Q6. What check belongs before *p in a function?
  • Answer: if (p != nullptr) before any *p or p->.

Lecture 24 Code Review

Trace the output

  • Q1. What prints? 
                
void swap(int* a, int* b) {
    int t = *a; *a = *b; *b = t;
}
int x = 3, y = 8;
swap(&x, &y);
std::cout << x << ' ' << y;
  • Answer: 8 3. The function swaps through the addresses, so the caller's x and y are exchanged.

Spot the bug

  • Q2. Why does this not compile? 
                
int x = 1, y = 2;
const int* p = &x;
*p = 10;          // line A
p  = &y;          // line B
  • Answer: Line A fails: p is a pointer to const, so the pointee cannot be written. Line B is fine — the pointer itself may be reseated.
