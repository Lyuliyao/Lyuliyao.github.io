Const Member Functions

Const member functions let objects promise read-only behavior through their interface.

Const Object

            
const string name = "Ada";
cout << name.length() << endl;
// name += "!";  // error
  • Const objects cannot be mutated.
  • They can call functions that promise not to mutate.
  • This idea extends to your own classes.

Const Member Function

            
struct Rectangle {
    double width;
    double height;

    double area() const {
        return width * height;
    }
};
  • The const after the parameter list is a promise.
  • The member function should not change the object.
  • Const objects can call const member functions.

Non-Const Member Function

            
struct Counter {
    int value;

    void bump() { ++value; }
    int get() const { return value; }
};
  • Mutating functions should not be marked const.
  • Observer functions usually should be const.
  • This makes interfaces safer.

Design Questions

Before writing code

  • What data changes over time?
  • What function owns each task?
  • What type communicates the intent?

Before debugging

  • What should happen on a tiny input?
  • Where can scope or lifetime end?
  • Which operation may access invalid memory?

Why Mark Member Functions const?

            
struct Counter {
    int value;

    int get() const {
        return value;
    }
};

const Counter c{5};
cout << c.get() << endl;
  • A const object can only call member functions that promise not to mutate it.
  • The promise appears after the parameter list: get() const.
  • Observer functions should usually be marked const.

Build Error: Mutating Inside const

            
struct Counter {
    int value;

    int bad_get() const {
        ++value;      // build error
        return value;
    }
};
  • The function promised not to modify the object.
  • Changing a member variable breaks that promise.
  • This is the member-function version of reference-to-const protection.

Trace it

  • Q1. What should you trace first? 
                
const vector<int> values = {1, 2, 3};
cout << values.size() << endl;
// values.push_back(4);  // error
  • Answer: A const vector can be inspected but not changed.

Fix the design

  • Q2. What design question matters here? 
                
struct Stopwatch {
    double seconds;

    double elapsed() const { return seconds; }
    void reset() { seconds = 0; }
};
  • Answer: elapsed can be const, but reset must be non-const.
