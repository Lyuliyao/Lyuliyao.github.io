Syntax Review

            
// Declaration:
return_type name(param_type, param_type);

// Definition:
return_type name(param_type param_name) {
    instructions
}

// Call:
name(argument);
  • A declaration may omit parameter names.
  • A definition needs parameter names if the body uses them.
  • A call supplies arguments, not parameter declarations.

void Return Type

            
void print(double d) {
    cout << "printing... " << d << endl;
    // return;  // optional at the end
}

// void x;  // build error
  • void has no values.
  • A return type of void means no value is returned.
  • A void function may still use plain return; to end the call.

Trace a Function Call

            
int my_max(int i, int j) {
    if (i > j) {
        return i;
    }
    return j;
}

int a = 3;
int b = 7;
int c = my_max(a, b);
  • Create a function scope containing parameters i and j.
  • Copy arguments into parameters: i = 3, j = 7.
  • Execute the body; return j ends the call and assigns 7 to c.

Return Values

            
int abs_val(int x) {
    if (x < 0) {
        return -x;
    }
    return x;
}
  • A return statement ends the function call.
  • The expression after return becomes the value of the call.
  • Code after a guaranteed return does not run.

Returned Values Can Be Ignored

            
int square(int n) {
    cout << "called ";
    return n * n;
}

square(2);          // prints, returned 4 is ignored
int y = square(3);  // prints, returned 9 is stored
cout << y << endl;
  • The function body still runs even if the returned value is not stored.
  • If the call is part of an assignment, the returned expression supplies the assigned value.
  • Ignoring return values is legal, but often a sign to check your intent.

One Call, At Most One Returned Value

            
int combine(int a, int b) {
    return a + b;
    return a * b;   // never runs
}

int value = combine(2, 5);  // value is 7
  • A return statement ends the function call immediately.
  • Later return statements run only on paths that actually reach them.
  • If you need several pieces of information, one plain return value is not enough.

Function Scope

            
int add_one(int x) {
    int y = x + 1;
    return y;
}

cout << y << endl;   // error
  • Parameters and local variables live in function scope.
  • They are created for each call.
  • They disappear when the call ends.

Function Scope Is Not main's Scope

            
void f();

int main() {
    int i = 0;
    f();
}

void f() {
    i = 1;   // build error
}
  • f's function scope is not nested inside main's scope.
  • A function body can use its parameters and its own local variables.
  • Global variables are an exception, but they are usually discouraged in PIC 10A code.

Undefined Behavior: Missing Return

            
int abs_val(int i) {
    if (i > 0) {
        return i;
    }
    if (i < 0) {
        return -i;
    }
}

int v = abs_val(0);  // undefined behavior
  • abs_val has return type int.
  • When i == 0, the body reaches the closing brace without returning.
  • Using the missing return value is undefined behavior.

Function Comments

            
/**
  Returns the larger of two integers.
  @param i the first integer
  @param j the second integer
  @return the larger of i and j
*/
int my_max(int i, int j);
  • Start with a short description of what the function does.
  • Use @param for each parameter.
  • Use @return when the return type is not void.

Pure Functions and Procedures

            
int square(int n) {
    return n * n;
}

void print_square(int n) {
    cout << n * n << endl;
}
  • A pure function returns a value and has no side effects.
  • A procedure has return type void and does something observable.
  • Some functions mix both, but simple designs are easier to reason about.

Mixed Function: Neither Category

            
string read_until_newline() {
    string s;
    while (cin.peek() != '\n') {
        s += cin.get();
    }
    cin.ignore();
    return s;
}
  • It returns a string, so it is not a procedure.
  • It changes the input buffer, so it is not a pure function.
  • Mixed functions can be useful, but document their side effects clearly.

main Is a Function

            
int main() {
    cout << "program starts here" << endl;
    return 0;
}
  • main has a return type, name, parameter list, and body.
  • When the program runs, execution begins inside main.
  • In this course, return 0; means the program finished successfully.

Lecture 12 Summary

Main ideas

  • A return statement ends the current function call.
  • Function scope is created for each call and then destroyed.
  • Function scope is not nested inside the caller's local scope.
  • main is also a function, and program execution starts there.
  • Function comments document how to use a function.
  • Pure functions, procedures, and mixed functions have different design tradeoffs.

Programming habits

  • Prefer returning computed values instead of printing from every helper.
  • Write comments for functions whose purpose is not obvious.
  • Check all paths in non-void functions.
  • Do not claim that one function call returns two values.

Lecture 12 Review Questions

Core ideas

  • Q1. What does return do?
  • Answer: It ends the call and optionally sends back a value.
  • Q2. What does void mean?
  • Answer: The function does not return a value.
  • Q3. Where do parameters live?
  • Answer: In the function scope for that call.
  • Q4. What is a pure function?
  • Answer: A value-returning function without side effects.
  • Q5. Can a declaration omit parameter names?
  • Answer: Yes. The parameter types are the important part of the declaration.

Reasoning

  • Q6. What is a procedure?
  • Answer: A void function that performs a side effect.
  • Q7. Why write function comments?
  • Answer: To document purpose, parameters, and return values.
  • Q8. Can a non-void function fall off the end safely?
  • Answer: No, that can cause undefined behavior.
  • Q9. How many values does one function call return?
  • Answer: At most one value.
  • Q10. Why is a function that returns a value and changes cin not pure?
  • Answer: Changing the input buffer is a side effect.

Lecture 12 Code Review

Missing return

  • Q1. What is wrong? 
                
int sign(int x) {
    if (x > 0) { return 1; }
    if (x < 0) { return -1; }
}
  • Answer: The path where x == 0 returns no value.

Scope confusion

  • Q2. Why does this fail? 
                
void f() {
    int local = 3;
}

cout << local << endl;
  • Answer: local exists only inside f.

Caller scope

  • Q3. Why does this fail? 
                
void f() {
    i = 1;
}

int main() {
    int i = 0;
    f();
}
  • Answer: f cannot access main's local variable i. Function scopes are separate.

Ignored return

  • Q4. What is printed? 
                
int f(int i) {
    cout << "f ";
    return i * i;
}

f(2);
cout << f(3) << endl;
  • Answer: f f 9. The first returned value is ignored, but the first call still prints.
