Print a Vector

            
void print(const vector<int>& v) {
    for (size_t i = 0; i < v.size(); ++i) {
        cout << v[i] << " ";
    }
    cout << endl;
}
  • Use const vector<int>& for read-only access.
  • The loop stays below v.size().
  • The function works for large vectors without copying.

Search

            
bool contains(const vector<int>& v, int target) {
    for (size_t i = 0; i < v.size(); ++i) {
        if (v[i] == target) { return true; }
    }
    return false;
}
  • Return as soon as the target is found.
  • If the loop finishes, the target was not present.
  • This is a common linear search pattern.

Summarize

            
int sum(const vector<int>& v) {
    int total = 0;
    for (size_t i = 0; i < v.size(); ++i) {
        total += v[i];
    }
    return total;
}
  • Initialize before the loop.
  • Visit each element once.
  • Return the computed value.

Why Passing by Value Can Hurt

            
void print_copy(vector<int> v) {
    for (size_t i = 0; i < v.size(); ++i) {
        cout << v[i] << " ";
    }
}

vector<int> big(10000, 8);
print_copy(big);
  • The function works, but the call copies all 10,000 elements.
  • That copy is unnecessary if the function only prints.
  • This motivates const vector<int>&.

Fix: Reference to Const

            
void print(const vector<int>& v) {
    cout << "[ ";
    for (size_t i = 0; i < v.size(); ++i) {
        cout << v[i] << " ";
    }
    cout << "]" << endl;
}
  • No vector copy is made.
  • The function promises not to modify the vector.
  • The same function can print ordinary and const vectors.

Lecture 17 Code Review

Trace it

  • Q1. What should you trace first? 
                
void print_reverse(const vector<int>& v) {
    for (size_t i = v.size(); i > 0; --i) {
        cout << v[i - 1] << " ";
    }
    cout << endl;
}
  • Answer: Trace i carefully; the element index is i - 1.

Fix the design

  • Q2. What design question matters here? 
                
int count_even(const vector<int>& v) {
    int count = 0;
    for (size_t i = 0; i < v.size(); ++i) {
        if (v[i] % 2 == 0) {
            ++count;
        }
    }
    return count;
}
  • Answer: The vector is read-only; the function returns the computed count.
