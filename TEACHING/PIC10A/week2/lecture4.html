Why std::string?

One char is too small

  • char letter = 'A'; stores one character.
  • A full name or sentence needs many characters together.
  • std::string stores a sequence of characters.

Typical uses

  • Names
  • Words and full sentences
  • User input that contains text
string first = "Ada"; string city = "Los Angeles";

Constructing std::string

            
#include <string>
using namespace std;

string s0;
string s1 = "hello";
string s2("Bruin");
string s3(5, '!');

cout << s0 << endl;
cout << s1 << endl;
cout << s2 << endl;
cout << s3 << endl;
  • Remember #include <string>.
  • s0 is the empty string.
  • s3(5, '!') makes "!!!!!".
  • String literals like "hello" are not the same type as string, but they can initialize one.

Concatenation

            
string first = "Grace";
string last = "Hopper";
string full = first + " " + last;

cout << full << endl;

full += "!";
cout << full << endl;
  • + builds a new string by joining pieces.
  • += appends onto an existing string.
  • Concatenation is a major reason strings feel more convenient than raw characters.
When you read first + " " + last, think “build a longer piece of text.”

Length, Size, and Empty

            
string s = "Hello world";

cout << s.length() << endl;
cout << s.size() << endl;
cout << boolalpha << s.empty() << endl;
  • length() and size() give the same answer for strings.
  • empty() tells you whether the string has zero characters.
  • We will soon care about the exact return type. For today, the main idea is “how many characters are there?”

Lecture 4 Summary

Main ideas

  • std::string stores text as a sequence of characters.
  • Construct strings with literals, direct initialization, or repeated characters.
  • Use + and += for concatenation.
  • Use length(), size(), and empty() for basic string facts.

Programming habits

  • Remember #include <string>.
  • Prefer meaningful variable names like first_name or message.
  • Think of concatenation as building a new string value.

Lecture 4 Review Questions

Core ideas

  • Q1. Why is char not enough when we want to store a full name?
  • Answer: Because char stores one character, while a full name needs a sequence of characters.
  • Q2. What header should you include before using std::string?
  • Answer: #include <string>
  • Q3. What does string s3(5, '!'); construct?
  • Answer: A string containing five exclamation points.
  • Q4. What is the difference between + and += for strings?
  • Answer: + creates a combined string expression; += appends onto an existing string variable.

Practice

  • Q5. What do length() and size() report for a string?
  • Answer: The number of characters in the string.
  • Q6. What does empty() check?
  • Answer: Whether the string has zero characters.
  • Q7. If first = "Ada" and last = "Lovelace", what does first + " " + last produce?
  • Answer: "Ada Lovelace"
  • Q8. Why is message a better variable name than x for a string?
  • Answer: Because it communicates what the text means.
