PIC 10A • Lecture 13

References I

References let a parameter refer to an existing variable instead of receiving a copy. We use swap to see why this matters.

Chapters 10.1-10.4 pass-by-value aliases swap

Review

Function calls

  • Arguments are assigned to parameters at the start of a call.
  • So far those assignments copied values.

Problem

  • Some functions need to modify caller variables.
  • Pass-by-value cannot do that.

Roadmap

Copy pass by value The function receives its own local parameter.
Alias reference parameter The parameter becomes another name for the argument variable.

Bad Swap

            
void bad_swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
}

int x = 5, y = 10;
bad_swap(x, y);
  • a and b are copies.
  • The copies swap successfully.
  • x and y do not change.

Reference Alias

            
int x = 5;
int& r = x;

r = 8;
cout << x << endl;   // 8
  • A reference is an alias for an existing variable.
  • Using the reference means using the original variable.
  • A reference must be initialized.

Good Swap

            
void good_swap(int& a, int& b) {
    int temp = a;
    a = b;
    b = temp;
}

int x = 5, y = 10;
good_swap(x, y);
  • a references x.
  • b references y.
  • Assignments through a and b update the originals.

Assignments Through References

            
int i = 1;
int& r = i;
r = 2;

int j = 9;
r = j;   // assigns 9 to i
  • The reference does not rebind to j.
  • The assignment changes the value being referenced.
  • Draw arrows to avoid confusion.

Reference Rules

            
int& good = i;
// int& bad;        // build error
// int& also_bad = 4; // not for now
  • A non-const reference must start by referencing a variable.
  • It cannot be left uninitialized.
  • It stays attached to the same storage location.

Trace: Why bad_swap Fails

            
void bad_swap(int a, int b) {
    int c = a;
    a = b;
    b = c;
}

int i = 0;
int j = 1;
bad_swap(i, j);
  • The call creates a function scope containing copies a and b.
  • The body swaps the copies correctly.
  • When the call ends, the copies disappear and i, j are unchanged.

Trace: Why good_swap Works

            
void good_swap(int& a, int& b) {
    int c = a;
    a = b;
    b = c;
}

int i = 0;
int j = 1;
good_swap(i, j);
  • a references i; b references j.
  • Assignments through a and b update the original variables.
  • The function still has local variable c, which disappears after the call.

Lecture 13 Summary

Main ideas

  • Pass-by-value copies arguments into parameters.
  • A reference is an alias for an existing variable.
  • Reference parameters can modify caller variables.
  • Assignments through a reference affect the referred-to variable.

Programming habits

  • Use reference parameters only when mutation is intended.
  • Draw arrows for reference examples.
  • Use distinct names while learning function calls.

Lecture 13 Review Questions

Core ideas

  • Q1. What is pass-by-value?
  • Answer: Copying the argument value into the parameter.
  • Q2. Why does bad_swap not work?
  • Answer: It swaps only local copies.
  • Q3. What is a reference?
  • Answer: An alias for an existing variable.
  • Q4. What does int& a mean in a parameter list?
  • Answer: The parameter is a reference to an int.

Reasoning

  • Q5. Can a reference be uninitialized?
  • Answer: No.
  • Q6. Does assigning to a reference rebind it?
  • Answer: No, it assigns to the referred-to variable.
  • Q7. Why does good_swap work?
  • Answer: Its parameters reference the caller variables.
  • Q8. What visual model helps with references?
  • Answer: Arrows from references to variables.

Lecture 13 Code Review

Copy problem

  • Q1. What prints after this call? 
                
void f(int x) {
    x = 100;
}

int n = 3;
f(n);
cout << n << endl;
  • Answer: It prints 3; f changed only its copy.

Reference update

  • Q2. What prints here? 
                
void f(int& x) {
    x = 100;
}

int n = 3;
f(n);
cout << n << endl;
  • Answer: It prints 100.
