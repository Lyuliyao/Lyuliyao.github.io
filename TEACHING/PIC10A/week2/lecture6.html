cin and the Input Buffer

Today is about what the program actually reads from the keyboard: cin >> variable, getline, ignore, get, and peek.

Why input matters now

  • Strings become much more useful when users can type them.
  • Input bugs often come from misunderstanding the buffer.

Roadmap

Extraction cin >> variable Reads one formatted value and may leave extra characters behind.
Lines getline(cin, s) Reads a whole line, including spaces.

What cin >> variable Does

  • If the buffer is empty, the program waits for input.
  • Leading whitespace is skipped.
  • Characters are consumed according to the target type.
  • Unused characters stay in the input buffer.
int reads digits double reads decimal digits string stops at whitespace char reads one character

Examples of Leftovers

Example 1

Buffer contains 90024UCLA\n

  • int zip; reads 90024
  • Leftover becomes UCLA\n

Example 2

Buffer contains 900.24U CLA\n

  • string s; reads 900.24U
  • Leftover becomes CLA\n

getline(cin, s)

            
string full_name;
getline(cin, full_name);
cout << full_name << endl;
  • getline reads everything up to the newline.
  • The newline is removed from the buffer, but not stored in the string.
  • Unlike cin >> s, spaces are allowed inside the result.

Why Mixing >> and getline Breaks

            
int age;
cin >> age;

string name;
getline(cin, name);
  • After cin >> age, the trailing newline is still in the buffer.
  • getline immediately sees that newline and reads an empty string.
  • The usual fix is cin.ignore(); before getline.

ignore, get, and peek

cin.ignore()

Discard one character, often the leftover newline.

cin.get()

Extract one character and return it.

cin.peek()

Look at the next character without removing it.

A good mental model: peek looks, get takes, and ignore throws away.

Lecture 6 Summary

Main ideas

  • cin >> variable reads formatted input and can leave leftovers in the buffer.
  • getline reads a full line, including spaces.
  • Mixing >> and getline requires care.
  • ignore, get, and peek manipulate the buffer in different ways.

Programming habits

  • Ask what is still left in the buffer after each read.
  • Use cin.ignore() before getline when a leftover newline is the issue.
  • Choose the input tool that matches the kind of data you want.

Lecture 6 Review Questions

Core ideas

  • Q1. What does cin >> s do when s is a string?
  • Answer: It reads characters up to the next whitespace and stores them in s.
  • Q2. What does getline(cin, s) read?
  • Answer: The whole line up to the newline.
  • Q3. Why can getline read an empty string after formatted extraction?
  • Answer: Because the leftover newline may still be sitting in the input buffer.
  • Q4. What does cin.ignore() do in its simplest form?
  • Answer: It removes one character from the buffer.

Buffer tools

  • Q5. What is the difference between cin.get() and cin.peek()?
  • Answer: get() extracts the character, while peek() only looks at it.
  • Q6. Which input tool should you use for a full name with spaces?
  • Answer: getline
  • Q7. What should you ask yourself after every input line?
  • Answer: What characters are left in the buffer now?
  • Q8. Does cin >> int_var always empty the whole buffer?
  • Answer: No. It only consumes the characters needed for the integer and leaves the rest.
