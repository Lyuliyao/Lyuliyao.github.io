Course Snapshot

What We Will Learn

  • No prior programming experience is assumed.
  • Basic principles of programming in C++.
  • Algorithmic and procedural problem solving.
  • Program design and development.
  • Data types, control structures, and functions.
  • Arrays, pointers, and an introduction to classes.
  • PIC 10A is the prerequisite for later PIC courses.

Why This Course Matters

  • You are not only learning syntax. You are learning how programs store data, make decisions, and evolve step by step.
  • These foundations show up again in later languages, later PIC courses, and larger software projects.
  • The goal is to make you more independent when learning new tools later.
By the end of PIC 10A, you should be able to read short C++ programs, explain what they do, and write small correct programs of your own.

Topics Roadmap

Part 1 to Part 3

  • Basics: hello world, errors, variables, types, arithmetic, cin/cout, math functions, const.
  • Strings, control flow, and vectors: string, indexing, loops, conditionals, input buffer issues, vector.
  • Functions and references: declarations, definitions, parameters, return values, pure functions, procedures, pass-by-value and pass-by-reference.

Part 4 and Part 5

  • Structs and classes: interfaces, member functions, constructors, initializer lists, public/private, default values.
  • Arrays and pointers: C-style arrays, pointer arithmetic, passing arrays to functions, and string literals.
  • Source organization: headers, .cpp files, overloading, and const correctness.
This class covers a lot of ground. The course is designed to move from small syntax to increasingly structured programs.

Now let's code

The first few C++ programs

We will start small: a program that does almost nothing, then one that prints text, then one that uses escape sequences correctly.

The Smallest Useful Program

            
int main() {
    return 0;
}
  • Every C++ program must have exactly one main function.
  • Execution begins in main.
  • Semicolons ; mark the end of statements.
  • return 0; means the program finished successfully.

Hello, World!

            
#include <iostream>

int main() {
    std::cout << "Hello, World!\n";
    return 0;
}
  • #include <iostream> gives us stream-based input and output.
  • std::cout is the standard output stream.
  • << sends data to that output stream.
  • \n prints a newline character.

A Shorter Version

            
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, World!" << endl;
    return 0;
}
  • using namespace std; lets us omit the std:: prefix.
  • endl inserts a newline and flushes the output buffer.
  • This version is shorter, but it introduces a namespace decision we will discuss carefully in class.

Backslash Escape Sequences

\n Newline
\t Tab
\\ Backslash
\" Double quote
These sequences let you print characters that would otherwise be interpreted specially inside a string. 
            
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, \nWorld!\n";
    cout << "This is a tab:\tSee?\n";
    cout << "She said, \"Hello!\"\n";
    cout << "This is a backslash: \\\n";
    return 0;
}

Quick Check

What will this program print?

              
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, \\nWorld!\\n";
    return 0;
    cout << "Hello, World!\n";
}

Notice Three Things

  • \\n prints the two characters \ and n, not a newline.
  • return 0; ends the program immediately.
  • The final cout line is unreachable.
