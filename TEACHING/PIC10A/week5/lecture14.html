References II

References can avoid copies and can allow mutation. const references give read-only access, which is essential for safe and efficient function parameters.

Chapter 10.5 and Chapter 12 intro const references parameter choices mutation

Review

References I

  • A non-const reference parameter can modify the caller variable.
  • This is useful but should be intentional.

Today

  • Use const references for read-only access.
  • Choose parameter types based on size and mutation.

Roadmap

Read-only const T& Avoid a copy while promising not to mutate.
Mutable T& Use only when the function should change the argument.

Reference to Const

            
void print_name(const string& name) {
    cout << name << endl;
    // name += "!";  // build error
}
  • The parameter can read the string.
  • The function cannot modify it through the reference.
  • This avoids copying large objects.

Choosing Parameter Types

            
int square(int n);
void print(const string& s);
void increment(int& n);
  • Small fundamental types are usually passed by value.
  • Large read-only objects are often passed by const reference.
  • Use non-const reference only for intentional mutation.

Output Parameters

            
void divide(int a, int b, int& q, int& r) {
    q = a / b;
    r = a % b;
}
  • A non-const reference can be an output parameter.
  • Use this sparingly because it hides outputs in arguments.
  • Name output parameters clearly.

Const Arguments

            
void print(const string& s);

const string course = "PIC 10A";
print(course);
print("hello");
  • A const reference can bind to const objects.
  • It can also bind to temporary values in many useful cases.
  • This makes printing and read-only helpers flexible.

Parameter Decision Table

            
// wants a copy:        T x
// wants read-only:    const T& x
// wants to mutate:    T& x
  • Ask whether the function needs to change the argument.
  • Ask whether copying is cheap or expensive.
  • Make the signature communicate the answer.

Printing Without Copying

            
void print(const string& s) {
    for (size_t i = 0; i < s.length(); ++i) {
        cout << s[i];
    }
    cout << endl;
}
  • The function only reads the string, so mutation should be forbidden.
  • The reference avoids copying every character into a new string object.
  • This same idea becomes very important for vectors.

Choose the Parameter Type

Read-only helper

              
bool starts_with_a(const string& s) {
    return s.length() > 0 && s[0] == 'a';
}

Mutating helper

              
void make_zero(int& n) {
    n = 0;
}
The signature should tell the reader whether the function copies, reads, or mutates its argument.

Lecture 14 Summary

Main ideas

  • const T& gives read-only access without copying.
  • T& communicates intentional mutation.
  • Small values can usually be passed by value.
  • Parameter types are part of function design.

Programming habits

  • Do not use non-const references just to avoid typing.
  • Let the signature reveal whether mutation can happen.
  • Prefer simple return values when there is only one output.

Lecture 14 Review Questions

Core ideas

  • Q1. What does const string& s promise?
  • Answer: The function will not modify s through that reference.
  • Q2. Why avoid copying large objects?
  • Answer: Copying can waste time and memory.
  • Q3. When is int n fine?
  • Answer: When the function only needs a small copied value.
  • Q4. When is int& n appropriate?
  • Answer: When the function should modify the caller variable.

Reasoning

  • Q5. Can a non-const reference bind to a const object?
  • Answer: No.
  • Q6. What is an output parameter?
  • Answer: A reference parameter used to send a value back by mutation.
  • Q7. What should signatures communicate?
  • Answer: Whether arguments are copied, read, or mutated.
  • Q8. Which is clearer for one computed result: return or output parameter?
  • Answer: Usually return.

Lecture 14 Code Review

Const violation

  • Q1. Why does this fail? 
                
void shout(const string& s) {
    s += "!";
}
  • Answer: The function promised read-only access with const string&.

Parameter choice

  • Q2. Which parameter type is better for read-only vector printing? 
                
void print(vector<int> v);
void print(const vector<int>& v);
  • Answer: const vector<int>& avoids copying and prevents mutation.
