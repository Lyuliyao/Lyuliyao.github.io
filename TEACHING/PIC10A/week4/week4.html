Section 9: Functions

Create a function 

          
#include <iostream>
using namespace std;
int myFunction() {
  cout << "I just got executed!";
  return 0;
}

int main() {
  myFunction(); // call the function
  cout << endl;
  return 0;
}

Call a Function 

          
#include <iostream>
using namespace std;
void myFunction() {
  cout << "I just got executed!" << endl;
}

int main() {
  myFunction(); // call the function
  cout << endl;
  myFunction(); // call the function
  cout << endl;
  myFunction(); // call the function
  cout << endl;
  return 0;
}

Common Errors

  • It is a build error to call a function before its declaration.
  • It is a build error (linking error) to call a function when it is not defined.

Build Error

Because function scope is more stricter than the scope introduced by curly braces, it is a build error for a function body to use a variable which is not a parameter and which has not been declared inside the function body.

Example 

          
#include <iostream>
using namespace std;
void f();
int main() {
    int x = 10;
    return 0;
}
void f() {
    x = 1;
}

Undefined Behavior

Suppose f is a function with a non-void return type and that v is a variable of that return type. If the function body of f does not return a value, then the statement v = f(); causes undefined behavior.

Pure functions and procedures

  • Functions which return a value but do not have side effects are called pure functions.
  • Procedures which do not return a value but may have side effects are called procedures.

Example 

              
#include <iostream>
using namespace std;
int add(int a, int b) { // pure function
    return a + b;
}
void printSum(int a, int b) { // procedure
    cout << "The sum is: " << (a + b) << endl;
}
int main() {
    int x = 3, y = 4;
    int sum = add(x, y); // calling pure function
    cout << "Sum from function: " << sum
    << endl;
    printSum(x, y); // calling procedure
    return 0;
}
