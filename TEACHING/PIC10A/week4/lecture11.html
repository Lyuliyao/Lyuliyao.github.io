Vectors

Strings are one special container. A vector is the general version: an expandable sequence of values all having the same type.

Chapter 11.1-11.3 dynamic sequence push_back / pop_back indexing

Review from Lecture 10

Loop connection

  • We know how to traverse a string by index.
  • We know how to reason about bounds carefully.

Why that matters

  • Vectors also have indices, sizes, and traversal patterns.
  • Many string ideas transfer directly.

Roadmap

Container vector<T> A sequence of elements, all of the same type T.
Operations push_back, pop_back, size, indexing The same core ideas as strings, generalized.

Declaring Vectors

            
#include <vector>
#include <string>

vector<int> scores;
vector<double> weights;
vector<string> names;
  • Include <vector>.
  • The type inside angle brackets is the element type.
  • Every element in one vector has the same type.

push_back, pop_back, and size()

            
vector<int> v;

v.push_back(10);
v.push_back(20);
v.push_back(30);

cout << v.size() << endl;

v.pop_back();
cout << v.size() << endl;
  • push_back adds one element at the end.
  • pop_back removes the last element.
  • size() reports how many elements are currently stored.

Indexing: [] and at()

            
vector<int> v;
v.push_back(4);
v.push_back(7);
v.push_back(9);

cout << v[1] << endl;
cout << v.at(2) << endl;
  • The same safety story from strings appears again.
  • v[pos] out of range is undefined behavior.
  • v.at(pos) out of range is a runtime error.

Traversing a Vector

            
vector<int> v;
v.push_back(4);
v.push_back(7);
v.push_back(9);

for (size_t i = 0; i < v.size(); ++i) {
    cout << v[i] << endl;
}
  • This pattern should feel familiar from strings.
  • The stopping rule is still “stay strictly below the size.”
  • size_t appears again because sizes and indices belong together.

Vectors of Strings

            
vector<string> roster;
roster.push_back("Ada");
roster.push_back("Grace");

cout << roster[0] << endl;
cout << roster[1].length() << endl;
  • Vectors and strings compose naturally.
  • Each element can itself be a richer type like string.

Lecture 11 Summary

Main ideas

  • vector<T> stores a sequence of values of type T.
  • push_back, pop_back, and size() manage a vector’s contents.
  • Vectors use indexing with the same bounds cautions as strings.
  • Loops are the standard way to traverse vectors.

Programming habits

  • Keep the element type in mind at all times.
  • Use i < v.size(), not <=, for indexed traversal.
  • Recognize that many string skills transfer directly to vectors.

Lecture 11 Review Questions

Core ideas

  • Q1. What does vector<int> mean?
  • Answer: A vector whose elements are integers.
  • Q2. What does push_back do?
  • Answer: It adds one element at the end.
  • Q3. What does pop_back do?
  • Answer: It removes the last element.
  • Q4. What does size() report?
  • Answer: The number of elements currently in the vector.

Bounds and traversal

  • Q5. What is the last valid index when v.size() == n?
  • Answer: n - 1
  • Q6. What is the safety difference between v[pos] and v.at(pos)?
  • Answer: at() checks bounds and [] does not.
  • Q7. Why do loops and vectors go together naturally?
  • Answer: Because loops let us process each element one by one.
  • Q8. Can a vector store strings instead of ints?
  • Answer: Yes. For example, vector<string>.

Lecture 11 Code Review

Bounds bug

  • Q1. What is suspicious here? 
                      
for (size_t i = 0; i <= v.size(); ++i) {
    cout << v[i] << endl;
}
  • Answer: The loop reaches i == v.size(), which is already out of bounds for indexing.
