Roadmap

Branching if, if-else, else if Run different blocks depending on a condition.
Logic &&, ||, ! Build larger boolean tests from smaller ones.

if

            
if (score >= 60) {
    cout << "pass" << endl;
}
  • The condition must be a boolean expression.
  • If the condition is true, the block runs.
  • If the condition is false, the block is skipped.

if-else and else if

            
if (x > 0) {
    cout << "positive" << endl;
} else if (x < 0) {
    cout << "negative" << endl;
} else {
    cout << "zero" << endl;
}
  • Conditions are checked from top to bottom.
  • The first true branch runs.
  • The final else catches everything not handled earlier.

Boolean Operators

&&

Both parts must be true.

||

At least one part must be true.

!

Negates a boolean value.

Example: a leap year test is naturally a composition of several smaller boolean ideas.

Scope Inside Blocks

            
if (true) {
    int x = 5;
    cout << x << endl;
}

cout << x << endl;
  • The x inside the block only exists inside that block.
  • Trying to use it outside gives a build error.
  • Scope questions become very important as soon as control flow appears.

Lecture 8 Summary

Main ideas

  • if runs code conditionally.
  • if-else and else if let us branch among cases.
  • Boolean operators combine simpler conditions into richer logic.
  • Variables declared inside a block only exist in that block.

Programming habits

  • Read conditions aloud in English.
  • Keep branches mutually clear and logically ordered.
  • Use braces consistently.

Lecture 8 Review Questions

Branching

  • Q1. What happens when an if condition is false?
  • Answer: Its block is skipped.
  • Q2. In an else if chain, how many branches run?
  • Answer: At most one.
  • Q3. What does && mean?
  • Answer: Both conditions must be true.
  • Q4. What does ! do?
  • Answer: It negates a boolean value.

Scope

  • Q5. Why can a variable declared inside an if block cause a build error later?
  • Answer: Because it goes out of scope when the block ends.
  • Q6. What does || mean?
  • Answer: At least one condition must be true.
  • Q7. Why is ordering important in an else if chain?
  • Answer: Because the first true branch wins.
  • Q8. Why are braces a good habit?
  • Answer: They make block boundaries explicit and prevent subtle mistakes.

Lecture 8 Code Review

Build error

  • Q1. Why does this fail to build? 
                      
if (score >= 90) { char grade = 'A'; }
else if (score >= 80) { grade = 'B'; }
  • Answer: grade is declared only inside the first block, so the second branch cannot see it.

Logic check

  • Q2. What is wrong with this? 
                      
if (x % 2 == 0) {
    cout << "even" << endl;
}
if (x % 2 != 0) {
    cout << "odd" << endl;
}
  • Answer: It builds and runs, but it is better expressed as if-else because the two cases are complementary.
