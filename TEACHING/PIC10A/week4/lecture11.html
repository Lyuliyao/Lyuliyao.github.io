Functions I

Functions let us name reusable pieces of code. We start with declarations, definitions, calls, parameters, arguments, and signatures.

Chapter 9.1 declarations definitions parameters

Review

From control flow

  • Branches and loops organize instructions inside one function.
  • Now we learn how to create more functions.

Why functions

  • They reduce repeated code.
  • They give names to ideas and make programs easier to test.

Roadmap

Interface declaration Tell the compiler that a function exists.
Implementation definition Give the function body that actually runs.

Function Declaration

            
int my_max(int a, int b);
void print_line(string text);
  • A declaration ends with a semicolon.
  • It gives the return type, name, and parameter types.
  • It is enough for the compiler to understand later calls.

Function Definition

            
int my_max(int a, int b) {
    if (a > b) {
        return a;
    }
    return b;
}
  • A definition includes the body in braces.
  • A definition also counts as a declaration.
  • The body runs only when the function is called.

Calling a Function

            
int bigger = my_max(7, 4);
print_line("done");
  • Calling means using the function.
  • Arguments are the values supplied at the call site.
  • The returned value can be stored or used in a larger expression.

Parameters vs Arguments

            
int square(int n) {
    return n * n;
}

int x = 5;
int y = square(x);
  • n is a parameter.
  • x is an argument.
  • At the start of the call, the argument value is copied into the parameter.

Function Signatures

            
double average(int total, int count);
void print_score(string name, int score);
  • A signature is the name plus the parameter types.
  • Return type is important, but it does not distinguish overloads later.
  • Signatures are how declarations and calls are matched.

Declarations and Calls Together

            
int my_max(int i, int j);
void print(double d);

int main() {
    int a = 3;
    int b = 7;

    int c = my_max(a, b);
    cout << c << endl;

    print(1.01);
    return 0;
}
  • The declarations let main call the functions before their definitions appear.
  • a, b, and 1.01 are arguments.
  • The declarations are a compact map of the helper functions used by main.

Definitions After main

            
int my_max(int i, int j) {
    if (i > j) {
        return i;
    }
    return j;
}

void print(double d) {
    cout << "value: " << d << endl;
}
  • The definition gives the body that runs during a function call.
  • Parameter names such as i, j, and d belong to their function definitions.
  • The return type controls whether the function call produces a usable value.

Lecture 11 Summary

Main ideas

  • Function declarations announce functions before they are used.
  • Function definitions provide the executable body.
  • Arguments are provided by calls; parameters live inside the function call.
  • Function signatures document the name and parameter types.

Programming habits

  • Put declarations before calls.
  • Use names that describe the purpose of the function.
  • Keep one function focused on one job.

Lecture 11 Review Questions

Core ideas

  • Q1. What does a function declaration tell the compiler?
  • Answer: The return type, name, and parameter types.
  • Q2. What does a function definition add?
  • Answer: The function body.
  • Q3. What is a function call?
  • Answer: An instruction to run the function.
  • Q4. What is a parameter?
  • Answer: A variable listed in the function definition/declaration.

Reasoning

  • Q5. What is an argument?
  • Answer: A value supplied in a function call.
  • Q6. Can a function be declared before it is defined?
  • Answer: Yes.
  • Q7. Why are function names useful?
  • Answer: They give a reusable idea a readable name.
  • Q8. What should you check when a call does not build?
  • Answer: Whether a matching declaration is visible before the call.

Lecture 11 Code Review

Missing declaration

  • Q1. Why does this fail? 
                
int main() {
    cout << triple(4) << endl;
}

int triple(int n) {
    return 3 * n;
}
  • Answer: triple is called before the compiler has seen a declaration.

Parameter names

  • Q2. What value is printed? 
                
int f(int x) {
    return x + 1;
}

int x = 10;
cout << f(3) << endl;
  • Answer: It prints 4. The parameter x inside f is separate from the outer x.
