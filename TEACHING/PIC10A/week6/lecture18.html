Read-Only Vector Parameter

            
double average_or_zero(const vector<int>& v) {
    if (v.empty()) {
        return 0.0;
    }

    int total = 0;
    for (size_t i = 0; i < v.size(); ++i) {
        total += v[i];
    }
    return static_cast<double>(total) / v.size();
}
  • Read-only vector functions should usually use const T&.
  • Avoid copying the whole vector.
  • Do not mutate what you only need to inspect.

Mutating Vector Parameter

            
void add_one(vector<int>& v) {
    for (size_t i = 0; i < v.size(); ++i) {
        ++v[i];
    }
}
  • Use non-const reference when mutation is the point.
  • The caller sees the changed vector.
  • Make mutation obvious in the function name.

Returning a Vector

            
vector<int> first_n_squares(int n) {
    vector<int> result;
    for (int i = 1; i <= n; ++i) {
        result.push_back(i * i);
    }
    return result;
}
  • Returning a vector is clear for newly created data.
  • Build the result locally.
  • Let the call site store the returned vector.

Parameter Choice: Search

            
bool contains(const vector<double>& v, double target) {
    for (size_t i = 0; i < v.size(); ++i) {
        if (v[i] == target) {
            return true;
        }
    }
    return false;
}
  • The vector is read-only, so const vector<double>& is appropriate.
  • The target is a small fundamental type, so passing by value is fine.
  • Returning bool is clearer than using an output parameter.

Parameter Choice: Sort-Like Mutation

            
void clamp_nonnegative(vector<int>& v) {
    for (size_t i = 0; i < v.size(); ++i) {
        if (v[i] < 0) {
            v[i] = 0;
        }
    }
}
  • The function intentionally changes the caller's vector.
  • That is why the parameter is vector<int>&, not const vector<int>&.
  • The function name should make the mutation unsurprising.

void append_total(vector<int>& v) {
    int total = 0;
    for (size_t i = 0; i < v.size(); ++i) {
        total += v[i];
