Recap: int Arithmetic

            
#include <iostream>
using namespace std;

int main() {
    int i = 3, j = 2;

    cout << i + j << endl;
    cout << i - j << endl;
    cout << i * j << endl;

    i = i + 3;
    j += 3;
    cout << i << " " << j << endl;

    i *= 2;
    cout << i << endl;
}
  • We have the usual arithmetic operators: +, -, *.
  • We also have update forms: +=, -=, *=.
  • j += 3; is shorthand for j = j + 3;.

++, --, /, and %

Increment and decrement

              
int i = 0;
cout << ++i << endl; // prints 1  (increment, then use)
cout << i   << endl; // still 1

i = 0;
cout << i++ << endl; // prints 0  (use, then increment)
cout << i   << endl; // now 1
Never use ++ or -- more than once on the same variable in a single expression — the order is not guaranteed.

Integer division

              
int i = 28, j = 10;
cout << i / j << endl; // 2
cout << i % j << endl; // 8

i = -28;
cout << i / j << endl; // -2
cout << i % j << endl; // -8

For int, division discards the fractional part.

Undefined behavior: dividing an int by 0 is undefined behavior. The same warning applies to % 0.

What Is an int?

Typical model

  • On almost all machines you will use here, an int uses 4 bytes, or 32 bits.
  • That usually means the range is about -2^31 to 2^31 - 1.
  • So int is not “all integers.” It is a fixed-size container for a limited range.

Why this matters

  • If a computation tries to go outside the allowed range, the result is a problem.
  • For this course, you should treat int overflow as dangerous and not something to rely on.
  • So the type is useful, but it is finite and machine-represented.
Good mental model: an int is a fast, fixed-width storage box for ordinary whole-number computations, not a perfect model of mathematics.

Range Matters

            
#include <iostream>
using namespace std;

int main() {
    int biggest = 2147483647;
    cout << biggest << endl;
    biggest += 1;
    cout << biggest << endl;
}
  • An int has a limited range, so arithmetic is not unlimited.
  • If a calculation tries to go beyond that range, you are in dangerous territory.
  • For this course, treat integer overflow as something to avoid, not something to rely on.
This is one reason we asked “what is an int?” instead of treating it like pure mathematics.

const Protects Meaning

            
#include <iostream>
using namespace std;

int main() {
    const int HOURS_PER_DAY = 24;
    const int MINUTES_PER_HOUR = 60;
    const int MINUTES_PER_DAY = HOURS_PER_DAY * MINUTES_PER_HOUR;
    cout << MINUTES_PER_DAY << endl;
}
  • A const variable must be initialized when declared.
  • Assigning to a const later gives a build error.
  • This helps eliminate unexplained “magic numbers.”
const int DAYS = 7; good for fixed facts

Two Common const Errors

Build error

Not initialized

              
const int HOURS_PER_DAY;

A const fundamental type must be initialized when declared.

Build error

Assigned later

              
const int HOURS_PER_DAY = 24;
HOURS_PER_DAY = 25;

Once initialized, that value cannot be changed.

std::cin Reads Input

            
#include <iostream>
using namespace std;

int main() {
    int i1, i2;
    cout << "Enter two integers: ";
    cin >> i1 >> i2;
    cout << "Their sum is " << i1 + i2 << endl;
}
  • cin >> variable; extracts input and stores it in the variable.
  • You can chain extractions: cin >> i1 >> i2;.
  • The input must match the expected type.
Later we will study the input buffer more carefully. For now, think of cin as feeding typed values into variables.

bool and Comparison Operators

            
int a = 5, b = 10;
cout << boolalpha;

cout << (a == b) << endl; // false — equal?
cout << (a != b) << endl; // true  — not equal?
cout << (a <  b) << endl; // true  — less than?
cout << (a >  b) << endl; // false — greater than?
cout << (a <= b) << endl; // true  — less than or equal?
cout << (a >= b) << endl; // false — greater than or equal?
  • bool stores true or false.
  • Every comparison returns a bool you can store or print.
  • boolalpha makes booleans print as words instead of 1/0.
  • Do not confuse = (assignment) with == (equality test).
These operators work on any numeric type — we will use them heavily with if and loops.

Lecture 2 Summary

Main ideas

Lecture 2 Review Questions

Integer rules

  • Q1. What is the difference between ++i and i++ when they appear inside a larger expression?
  • Answer: ++i increments first and then yields the new value; i++ yields the old value and increments afterward.
  • Q2. What do 28 / 10 and 28 % 10 evaluate to?
  • Answer: 2 and 8.
  • Q3. What kind of problem occurs if you do i / 0 for an int?
  • Answer: Undefined behavior.
  • Q4. Why is it useful to remember that an int has a finite range?
  • Answer: Because machine integers are limited, so overflow and out-of-range computations are real problems.

Constants, input, and logic

  • Q5. Why is const int HOURS = 24; better than repeatedly typing 24?
  • Answer: It gives the value a meaningful name and prevents accidental reassignment.
  • Q6. What does cin >> a >> b; do?
  • Answer: It reads two inputs from the user into a and b.
  • Q7. What is the difference between = and ==?
  • Answer: = assigns a value, while == compares two values.
  • Q8. Write one expression that checks whether an integer n is even.
  • Answer: n % 2 == 0

Lecture 2 Code Review

Trace the output

  • Q1. This is adapted from textbook 5.3.5(1). What is the output? 
                      
int a = 2;
int b = 3;

cout << a << " " << b << endl;

a += 4;
b *= 5;

cout << a << " " << b << endl;

b %= a;

  • Answer: The output is 2 3, then 6 15, then 2 3, then 0 3.

Classify the problem

  • Q2. Is this a build error, runtime error, or undefined behavior? 
                      
const int A_CONSTANT;
cout << A_CONSTANT << endl;
  • Answer: Build error. A const int must be initialized when it is defined.
  • Q3. Is this a build error, runtime error, or undefined behavior? 
                      
int i = 8;
cout << i / 0 << endl;
  • Answer: Undefined behavior, because an int divided by 0 is undefined.
