while Loops

A loop runs code repeatedly. Today we focus on while: loop conditions, counting patterns, sentinel loops, do while, break, continue, and common infinite-loop mistakes.

Control flow so far

  • if chooses whether a block runs.
  • Conditions are boolean expressions.

New idea

  • while keeps revisiting the same block while the condition stays true.

Roadmap

Anatomy while (condition) { ... } Check first, then maybe run the body.
Risk Infinite loops Usually caused by a condition that never becomes false.
Control break and continue Sometimes a loop needs to leave early or skip one iteration.

The Basic Pattern

            
int i = 0;

while (i < 5) {
    cout << i << endl;
    ++i;
}
  • Initialize loop state first.
  • Test the condition before each iteration.
  • Update the state inside the loop body.

Counting and Accumulation

            
int i = 1;
int sum = 0;

while (i <= 4) {
    sum += i;
    ++i;
}
  • Loops often update more than one variable.
  • Here, i tracks position and sum tracks a running answer.

Sentinel Loops

            
char response = 'y';

while (response != 'n') {
    cout << "Continue? ";
    response = cin.get();
    cin.ignore();
}
  • The loop stops when a special value appears.
  • That special value is the sentinel.
  • Input and loops naturally combine.

do while: Check After

            
int number;

do {
    cout << "Enter a positive number: ";
    cin >> number;
} while (number <= 0);
  • The body runs once before the condition is checked.
  • This is useful when the loop must ask at least one time.
  • The while line ends with a semicolon.

break: Leave Early

            
int x;

while (true) {
    cout << "Enter a number (-1 to stop): ";
    cin >> x;

    if (x == -1) {
        break;
    }

    cout << x * x << endl;
}

cout << "done" << endl;
  • break exits the nearest loop immediately.
  • Use it when the stopping condition is found inside the body.
  • After break, execution continues after the loop.

continue: Skip the Rest

            
int i = 0;

while (i < 10) {
    ++i;
    if (i % 2 == 1) {
        continue;
    }
    cout << i << " ";
}
  • continue skips the rest of the current iteration.
  • The loop then checks its condition again.
  • In a while loop, be careful not to skip the update.

How Infinite Loops Happen

Common causes

  • Forgetting the update
  • Updating the wrong variable
  • Using a condition that can never become false

Debugging question

Which variable is supposed to change so that the condition eventually becomes false?

Lecture 8 Summary

Main ideas

  • while repeats while its condition is true.
  • Loop state must be initialized before the loop and updated during the loop.
  • Sentinel loops stop when a special value is seen.
  • do while runs the body once before checking the condition.
  • break exits a loop; continue skips to the next check.
  • Infinite loops usually come from broken state updates.

Programming habits

  • Trace loop variables line by line.
  • Check whether the condition can eventually become false.
  • Use break and continue only when they make the loop easier to read.
  • Keep loop bodies small enough to reason about.

Lecture 8 Review Questions

Loop basics

  • Q1. When does a while loop check its condition?
  • Answer: Before each iteration.
  • Q2. What three ingredients appear in most loops?
  • Answer: Initialization, a condition, and an update.
  • Q3. What is a sentinel loop?
  • Answer: A loop that stops when a special stopping value appears.
  • Q4. When is a do while loop useful?
  • Answer: When the body should run at least once.

Loop control

  • Q5. What does break do inside a loop?
  • Answer: It exits the nearest loop immediately.
  • Q6. What does continue do inside a loop?
  • Answer: It skips the rest of the current iteration and goes back to the loop condition.
  • Q7. Can a while loop body run zero times?
  • Answer: Yes, if the condition is false immediately.
  • Q8. What is the classic cause of an infinite loop?
  • Answer: The condition never becomes false.

Lecture 8 Code Review

Infinite loop

  • Q1. What is wrong with this loop? 
                      
int i = 0;
while (i < 10) {
    cout << i << endl;
}
  • Answer: i never changes, so the condition stays true forever. The loop is infinite.

Trace the result

  • Q2. What is printed? 
                      
int i = 3;
while (i > 0) {
    cout << i << " ";
    --i;
}
  • Answer: 3 2 1

More Code Review

Post-test loop

  • Q3. How many times can this loop run? 
                      
int x = 5;
do {
    cout << x << endl;
} while (x < 0);
  • Answer: Once. A do while checks the condition after the body.

Skipped update

  • Q4. Why is this dangerous? 
                      
int i = 0;
while (i < 5) {
    if (i == 2) {
        continue;
    }
    ++i;
}
  • Answer: When i is 2, continue skips ++i, so the loop gets stuck.
