Vectors

std::vector<T> is an adaptive container that can grow and shrink. We learn how to construct one, index into it, and pass it efficiently to a function.

Roadmap

Class template std::vector<T> Pick any type T to get a class.

What Is a Vector?

            
#include <vector>
using namespace std;

vector<int>    vi;     // empty vector of int
vector<double> vd;     // empty vector of double
vector<string> vs;     // empty vector of string
  • std::vector is a class template.
  • Pick a type T and vector<T> is a class.
  • An instance holds a growable sequence of T values.

Constructors

            
vector<int> a;                 // empty
vector<int> b(5);              // {0, 0, 0, 0, 0}
vector<int> c(5, 8);           // {8, 8, 8, 8, 8}
vector<int> d{1, 2, 3, 4};     // {1, 2, 3, 4}
  • A size_t n: n value-initialized elements (0 for fundamental types).
  • (n, t): n copies of t.
  • Brace list: exactly those elements.

Growing and Shrinking

            
vector<int> v;
v.push_back(10);   // {10}
v.push_back(20);   // {10, 20}
v.push_back(30);   // {10, 20, 30}
v.pop_back();      // {10, 20}

cout << v.size() << endl;   // 2
cout << v.empty() << endl;  // 0
v.clear();                  // {}
  • push_back appends a new element at the end.
  • pop_back removes the last element.
  • size returns a size_t; empty returns a bool.

Indexing

            
vector<int> v{10, 20, 30};

cout << v[0] << endl;     // 10
cout << v.at(2) << endl;  // 30

cout << v[5];             // undefined behavior
cout << v.at(5);          // runtime error
  • Valid indices are 0 .. size()-1.
  • v[pos] with pos >= size() is undefined behavior.
  • v.at(pos) throws a runtime error instead.

resize and reserve

            
vector<int> v{1, 2, 3};

v.resize(5);        // {1, 2, 3, 0, 0}
v.resize(7, 9);     // {1, 2, 3, 0, 0, 9, 9}
v.resize(2);        // {1, 2}
  • resize(n) changes size() to n.
  • New elements are value-initialized, or copies of the second argument.
  • Shrinking drops the last elements.

Printing a Vector

            
void print(vector<int> v) {
    if (v.size() == 0) { cout << "{ }" << endl; return; }

    cout << "{ ";
    for (size_t i = 0, N = v.size() - 1; i < N; ++i) {
        cout << v[i] << ", ";
    }
    cout << v[v.size() - 1] << " }" << endl;
}
  • There is no built-in operator<< for vectors.
  • We write our own print function.
  • The parameter is currently passed by value.

The Hidden Cost

            
int main() {
    vector<int> w(10000, 8);
    print(w);          // copies 10,000 ints into v
    return 0;
}
  • The argument-to-parameter assignment is vector<int> v = w.
  • That copies every element, then discards the copy.
  • Printing should not first duplicate the data.

First Fix: Reference

            
void print(vector<int>& v) {
    // ... same body ...
}

int main() {
    vector<int> w(10000, 8);
    print(w);    // no copying
    return 0;
}
  • vector<int>& v = w aliases w; no copy.
  • But this signature claims print may mutate.
  • And it cannot accept a const vector.

Better: Reference to Const

            
void print(const vector<int>& v) {
    if (v.size() == 0) { cout << "{ }" << endl; return; }
    cout << "{ ";
    for (size_t i = 0, N = v.size() - 1; i < N; ++i) {
        cout << v[i] << ", ";
    }
    cout << v[v.size() - 1] << " }" << endl;
}
  • No copying ever happens.
  • The signature promises v will not be mutated.
  • It accepts both vector<int> and const vector<int> arguments.

Read-Only Permission

A reference to const is like sharing a Google Doc with read-only permission: the reader can see your work, cannot accidentally change it, and you do not have to make a copy first.

Lecture 15 Summary

Main ideas

  • std::vector<T> is a growable container of T.
  • Use push_back, pop_back, size, at, resize.
  • Indexing past size()-1 is undefined behavior.
  • Pass large vectors by const reference.

Programming habits

  • Default to const vector<T>& for read-only access.
  • Use at while learning to catch out-of-range bugs.
  • Loop with size_t i = 0; i < v.size(); ++i.

Lecture 15 Review Questions

Core ideas

  • Q1. What does v.push_back(x) do?
  • Answer: Appends x at the end and increases size() by one.
  • Q2. What is the difference between v[i] and v.at(i)?
  • Answer: at throws a runtime error if i is out of range; [] is undefined behavior.
  • Q3. What does vector<int> v(3) contain?
  • Answer: Three zeros.
  • Q4. What does vector<int> v{3} contain?
  • Answer: One element with value 3.

Reasoning

  • Q5. Why is void print(vector<int> v) bad for large vectors?
  • Answer: It copies every element into the parameter.
  • Q6. Why is void print(vector<int>& v) still imperfect?
  • Answer: It claims mutation and rejects const arguments.
  • Q7. Why does const vector<int>& solve both problems?
  • Answer: No copy, no mutation, accepts const and non-const arguments.
  • Q8. What is the type of v.size()?
  • Answer: size_t (an unsigned integer type).
