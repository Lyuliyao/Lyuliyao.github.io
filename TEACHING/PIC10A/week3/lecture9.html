for Loops

The for loop is the standard tool when a loop variable changes in a regular way. Today we focus on counting, bounds, step sizes, string traversal, and loop-variable scope.

Chapter 8.5 counting loops bounds string traversal

Review

From while

  • A loop needs initialization, a condition, and an update.
  • A bad update can create an infinite loop.

Why for

  • Counting loops repeat with a predictable update.
  • for keeps the loop machinery in one header.

Roadmap

Anatomy for (init; test; update) Read the header from left to right.
Patterns count, step, traverse Most early for loops fit one of these shapes.

Basic Counting Form

            
for (int i = 0; i < 5; ++i) {
    cout << i << endl;
}
  • Initialization runs once before the loop starts.
  • The condition is checked before each iteration.
  • The update runs after the body each time.

Counting Ranges

            
for (int i = 2; i < 6; ++i) {
    cout << i << " ";
}

// prints: 2 3 4 5
  • The starting value is included.
  • The loop stops before the first value that makes the condition false.
  • Translate the header into a list of values before trusting it.

Step Sizes and Countdown

            
for (int i = 0; i <= 10; i += 2) {
    cout << i << " ";
}

for (int j = 5; j > 0; --j) {
    cout << j << " ";
}
  • The update can add more than one.
  • A countdown usually uses --j and a decreasing condition.
  • The condition and update must move in compatible directions.

Traversing a String

            
string s = "bruins";

for (size_t i = 0; i < s.length(); ++i) {
    cout << i << ": " << s[i] << endl;
}
  • Valid indices are 0 through s.length() - 1.
  • Use i < s.length(), not i <= s.length().
  • size_t is the type used by string sizes.

Loop Variable Scope

            
for (int i = 0; i < 3; ++i) {
    cout << i << endl;
}

cout << i << endl;   // build error
  • A loop variable declared in the header belongs to the for statement.
  • After the loop ends, that i is out of scope.
  • Declare a variable before the loop only if code after the loop needs it.

Table of Squares

            
cout << "n  n^2" << endl;
cout << "------" << endl;

for (int n = 1; n <= 6; ++n) {
    cout << n << "  " << n * n << endl;
}
  • This is a regular counting loop, so for is the natural choice.
  • The loop variable n is also the data being processed.
  • The body computes one row of the table per iteration.

Count Matches

            
string s = "mississippi";
int count = 0;

for (size_t i = 0; i < s.length(); ++i) {
    if (s[i] == 's') {
        ++count;
    }
}

cout << count << endl;
  • This combines string traversal, if, and an accumulator.
  • count must be declared before the loop because it is needed after the loop.
  • Each iteration asks one small question: is this character a match?

Lecture 9 Summary

Main ideas

  • for packages initialization, condition, and update together.
  • Counting loops are easiest to debug by listing the values of the loop variable.
  • String traversal requires staying strictly below the length.
  • Loop variables declared in the header have limited scope.

Programming habits

  • Trace small loops by hand.
  • Check the start, stop, and update together.
  • Prefer ++i or clear step updates over hidden state changes.

Lecture 9 Review Questions

Loop headers

  • Q1. What are the three pieces of a for loop header?
  • Answer: Initialization, condition, and update.
  • Q2. When does the update run?
  • Answer: After each execution of the loop body.
  • Q3. What does ++i do?
  • Answer: It increases i by one.
  • Q4. Why use size_t for string indices?
  • Answer: String lengths are reported as size_t.

Reasoning

  • Q5. What values occur in for (int i = 1; i < 4; ++i)?
  • Answer: 1, 2, and 3.
  • Q6. What is the last valid index of a string of length n?
  • Answer: n - 1.
  • Q7. Can a loop variable declared in a for header be used after the loop?
  • Answer: No.
  • Q8. What should you inspect first for too many iterations?
  • Answer: The loop bounds and update.

Lecture 9 Code Review

Off-by-one

  • Q1. What is suspicious here? 
                
for (size_t i = 0; i <= s.length(); ++i) {
    cout << s[i] << endl;
}
  • Answer: The loop reaches i == s.length(), which is one past the last valid index.

Wrong variable

  • Q2. What does this actually print? 
                
for (int i = 0; i < 3; ++i) {
    cout << 3 << endl;
}
  • Answer: It prints 3 three times, not the values of i.
