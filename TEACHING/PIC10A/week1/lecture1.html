Why the compiler cares

  • The type tells the compiler what kind of value is stored.
  • The type determines which operations make sense.
  • Even things we already used have types: std::cout, string literals, and endl.
Thinking “what is the type of this thing?” will become one of your best debugging habits.

Variables Are Named Storage

What a variable gives us

  • A place to store a value.
  • A name we can use later.
  • A type that tells us how to interpret the stored bits.
  • For non-const variables, the value can change.

Declaring and Assigning

            
#include <iostream>
using namespace std;

int main() {
    int i;
    i = 1;
    cout << i << endl;
    return 0;
}
  • int i; declares and defines a variable named i.
  • i = 1; assigns the value 1 to i.
  • cout << i prints the value stored in the variable, not the string literal "i".
After line 5 i exists
After line 6 i = 1

Two Early Build Errors

Build error

Use before declaration

              
int main() {
    j = 1;
    return 0;
}

j does not exist yet.

Build error

Redefinition

              
int main() {
    int i;
    int i;
    return 0;
}

Same name, same scope, declared twice.

Build errors are good news compared to hidden bugs: the compiler caught a problem before the program ran.

The Assignment Operator

            
#include <iostream>
using namespace std;

int main() {
    int i, j, k;
    i = 1; j = 2; k = 3;
    cout << i << " " << j << " " << k << endl;
    i = j;
    cout << i << " " << j << " " << k << endl;
    i = j + k;
    cout << i << " " << j << " " << k << endl;
}
  • = means “take the value on the right and store it in the variable on the left.”
  • The left side must be something assignable, usually a variable.
  • i = j + k; updates i. It does not change j or k.
i = j i = j + k = is not equality

Initialization and Undefined Behavior

            
#include <iostream>
using namespace std;

int main() {
    int ub;
    cout << ub << endl;
    ub = 1;
    return 0;
}
  • The first assignment to a variable is called initialization.
  • Reading a variable before initialization gives undefined behavior.
  • Undefined behavior is dangerous because the program may appear to work sometimes.
Do not rely on “it printed 0 on my machine.” Undefined behavior means the language gives no promise.

Naming Rules and Conventions

Rules

  • Names may start with a letter or underscore.
  • After that, letters, digits, and underscores are allowed.
  • Names are case-sensitive: score and Score differ.
  • 1score is illegal because it starts with a digit.

Conventions

  • Use names that describe the value: minutes_per_day, num_students.
  • Many codebases use lower_snake_case or lowerCamelCase.
  • For constants, names like HOURS_PER_DAY are common.
Good names reduce comments because the code explains itself.

int Arithmetic

            
#include <iostream>
using namespace std;

int main() {
    int i = 3, j = 2;

    cout << i + j << endl;
    cout << i - j << endl;
    cout << i * j << endl;

    i = i + 3;
    j += 3;
    cout << i << " " << j << endl;

    i *= 2;
    cout << i << endl;
}
  • We have the usual arithmetic operators: +, -, *.
  • We also have update forms: +=, -=, *=.
  • j += 3; is shorthand for j = j + 3;.

Lecture 1 Summary

Main ideas

  • Everything has a type, and variables are named storage.
  • = assigns a value to a variable; it is not a test for equality.
  • Initialization matters, because reading an uninitialized variable is undefined behavior.
  • int supports basic arithmetic and update operators such as +=.

Programming habits

  • Initialize variables before using them.
  • Use names that reveal the meaning of the value.
  • Treat build errors as useful feedback from the compiler.
  • Trace variable values line by line when code feels confusing.
If you can trace how a variable changes from line to line, you understand the core of today's lecture.

Lecture 1 Review Questions

Core ideas

  • Q1. How do you declare an int called my_int, and how do you make it store 8?
  • Answer: int my_int; my_int = 8;
  • Q2. What kind of error do you get if you use a variable before declaring it?
  • Answer: A build error.
  • Q3. What is the difference between = and ==?
  • Answer: = assigns a value; == compares two values.
  • Q4. What happens if you print an int before initializing it?
  • Answer: That is undefined behavior.

Computation and style

  • Q5. If i = 3 and j = 2, what does j += 3; do?
  • Answer: It updates j to 5; it is shorthand for j = j + 3;.
  • Q6. Why is 1score not a valid variable name?
  • Answer: Because variable names cannot begin with a digit.
  • Q7. What is the benefit of a descriptive name like minutes_per_day?
  • Answer: It makes the code easier to read and reduces guesswork about what the value means.
  • Q8. If you see a build error caused by a misspelled variable name, what does that tell you?
  • Answer: The compiler found a name it cannot match to a declared variable in that scope.

  • Answer: It builds, but printing score before initialization is undefined behavior.
