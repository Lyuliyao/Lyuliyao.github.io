How to Choose Parameter Types

For any type T, a parameter can be T, T&, or const T&. Two questions decide which one to write.

Chapter 12 pass-by-value const T& const correctness

Review

What we know

  • T copies the argument.
  • T& aliases and may mutate.
  • const T& aliases for read-only access.

Today

  • A simple decision procedure.
  • When to worry about the cost of a copy.
  • What const correctness buys.

Two Questions

Q1 Mutate the argument? If yes → T&.
Q2 Avoid copying? If yes (and not mutating) → const T&.
Otherwise Use T The simplest choice.

The Decision in Code

            
if (mutate_argument) {
    // parameter type: T&
}
else if (avoid_copying) {
    // parameter type: const T&
}
else {
    // parameter type: T
}
  • Mutation has the strongest requirement; check it first.
  • If we cannot mutate, we still might want to skip a copy.
  • If neither matters, plain T is fine.

When Is Copying Cheap?

Fundamental types

  • int, double, char, bool, …
  • Always 1–8 bytes.
  • Copying is essentially free.

Class types

  • std::string can hold millions of characters.
  • std::vector<T> can hold many elements.
  • Copying may be expensive.

Rule of Thumb

            
if (mutate_argument) {
    // T&
}
else if (class_type) {
    // const T&
}
else {
    // T
}
  • Class type, no mutation → const T&.
  • Fundamental type, no mutation → T.
  • Mutation needed → T& regardless.

Examples That Match the Rule

            
int square(int n);                       // T
void print(const string& s);             // const T&
void print(const vector<int>& v);        // const T&
void increment(int& n);                  // T&
void sort(vector<int>& v);               // T&
  • square: small fundamental type, no mutation.
  • print: large class type, no mutation.
  • increment, sort: mutation is the whole point.

Const Correctness

            
int sum(const vector<int>& v) {
    int s = 0;
    for (size_t i = 0, N = v.size(); i < N; ++i) s += v[i];
    return s;
}

void print_sum(const vector<int>& v) {
    cout << sum(v) << endl;
}
  • print_sum calls sum with its const reference.
  • If sum dropped the const, print_sum would not compile.
  • Being const correct lets your code be reused by const-correct callers.

What If sum Were Not Const Correct?

            
int sum(vector<int>& v);                 // missing const

void print_sum(const vector<int>& w) {
    cout << sum(w) << endl;               // build error
}
  • The argument-to-parameter assignment would be vector<int>& v = w.
  • w is a reference to const, so this is rejected.
  • The author of sum broke the author of print_sum.

A Note on const T

            
void f(int i) {}
void f(const int i) {}   // build error: redefinition
  • A const on a non-reference parameter is invisible to callers.
  • The compiler treats the two declarations as the same function.
  • Use const T& when you mean read-only.

Worked Example: sort

            
void sort(vector<int>& v);
  • Q1: do we mutate? Yes — sorting reorders the elements.
  • Therefore the parameter must be vector<int>&.
  • Neither const vector<int>& nor vector<int> would work.

Worked Example: find

            
size_t find(const vector<double>& v, double target);
  • Q1: mutate v? No.
  • Q2: avoid copying v? Yes, vectors can be large.
  • So v is const vector<double>&; target is a small double, pass by value.

Lecture 16 Summary

Main ideas

  • Ask: do I mutate? do I want to avoid copying?
  • Mutate → T&.
  • Class type, read-only → const T&.
  • Fundamental type, read-only → T.

Programming habits

  • Be const correct so others can call your function.
  • Do not write const T on non-reference parameters.
  • Make signatures advertise intent.

Lecture 16 Review Questions

Core ideas

  • Q1. Best parameter type to sort a vector<int> in place?
  • Answer: vector<int>&.
  • Q2. Best for find on a vector<double>?
  • Answer: const vector<double>& for the vector; double for the target.
  • Q3. What does "const correct" mean?
  • Answer: Marking parameters const whenever the function does not mutate them.
  • Q4. Why prefer const T& for class types?
  • Answer: Avoids a potentially expensive copy and forbids mutation.

Reasoning

  • Q5. Why use int rather than const int&?
  • Answer: Copying an int is essentially free; a reference adds no benefit.
  • Q6. Why does dropping const on a parameter break callers?
  • Answer: const arguments can no longer bind to the parameter.
  • Q7. Is void f(const int) different from void f(int)?
  • Answer: No — declaring both is a redefinition error.
  • Q8. What does a parameter signature communicate?
  • Answer: Whether the function copies, reads, or mutates the argument.

Lecture 16 Code Review

Why does this fail?

  • Q1. 
                
int sum(vector<int>& v);

void print_sum(const vector<int>& w) {
    cout << sum(w) << endl;
}
  • Answer: sum is not const correct: a const argument cannot bind to a non-const reference parameter.

Pick the parameter type

  • Q2. A function appends a string to every element of a vector<string>. What parameter types?
  • Answer: vector<string>& for the vector (mutated), const string& for the suffix (read-only, class type).
