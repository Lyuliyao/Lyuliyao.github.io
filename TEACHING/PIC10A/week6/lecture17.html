What #include Does

            
// audio.hpp
#include <iostream>

/** Does an awesome audio thing. */
void do_awesome_audio_thing();
  • #include "audio.hpp" is replaced by the entire contents of that file.
  • Use "..." for project files, <...> for standard library.
  • It is a textual paste, nothing more.

A Multi-File Project

audio.hpp

              
#include <iostream>
void do_awesome_audio_thing();

audio.cpp

              
#include "audio.hpp"

void do_awesome_audio_thing() {
    std::cout << "audio thing" << std::endl;
}

Headers Including Headers

            
// video.hpp
#include <iostream>
#include "audio.hpp"   // videos need audio
#include "image.hpp"   // videos need images

void do_awesome_video_thing();
  • A header includes other headers it depends on.
  • The same declaration may end up appearing many times.
  • Multiple declarations of the same function are fine in C++.

main.cpp Pulls It All Together

            
// main.cpp
#include <iostream>
#include "audio.hpp"
#include "image.hpp"
#include "video.hpp"

int main() {
    do_awesome_audio_thing();
    do_awesome_image_thing();
    do_awesome_video_thing();
    return 0;
}
  • main.cpp sees declarations from each header.
  • It does not see the definitions in the .cpp files.
  • The linker connects calls to definitions later.

How to Organize Your Code

Put in the header

  • Function comments (the "documentation").
  • Function declarations.
  • #includes the declarations need.

Put in the cpp

  • Function definitions.
  • #include "matching_header.hpp".
  • Optionally using namespace std;.

Never using namespace in a Header

            
// fs.hpp -- BAD
#include <vector>
using namespace std;          // forced on every includer
void print(const vector<int>& v);
  • A using in a header leaks into every file that includes it.
  • Callers may not want std brought in.
  • Write fully qualified names like std::vector instead.

Example: functions.hpp

            
// functions.hpp
#include <iostream>
#include <vector>

/**
 Prints the elements of a vector of ints
 separated by a comma and a space and
 enclosed by braces {}.
 @param v: the vector of ints to print
*/
void print(const std::vector<int>& v);
  • Declaration only, plus the comment.
  • Fully qualified std::vector.
  • The header is the public face of print.

Example: functions.cpp

            
// functions.cpp
#include "functions.hpp"
using namespace std;

void print(const vector<int>& v) {
    if (v.size() == 0) { cout << "{ }" << endl; return; }
    cout << "{ ";
    for (size_t i = 0, N = v.size() - 1; i < N; ++i) {
        cout << v[i] << ", ";
    }
    cout << v[v.size() - 1] << " }" << endl;
}
  • Definition only.
  • Includes its own header to be sure declarations agree.
  • using namespace std; is okay inside a .cpp.

Example: main.cpp

            
// main.cpp
#include "functions.hpp"
using namespace std;

int main() {
    vector<int> w{1, 2, 3};
    print(w);
    return 0;
}
  • main.cpp only includes the header.
  • It needs only the declaration to call print.
  • The definition is found at link time.

Compile, Then Link

Compile

  • Each .cpp is compiled separately into an object file.
  • Headers are not compiled; they are pasted in.
  • Each compilation needs declarations for everything it calls.

Link

  • Object files are stitched into one executable.
  • Each call site is matched with a definition.
  • Missing definitions become linker errors.

Does main.cpp Know functions.cpp?

No. main.cpp only sees the header. The compiler is happy because the declaration is there. The linker fills in the definition from the compiled functions.cpp later.

Lecture 17 Summary

Main ideas

  • Headers hold comments and declarations.
  • Cpp files hold definitions.
  • #include is a textual paste.
  • Compilation produces object files; linking joins them.

Programming habits

  • Each fs.cpp includes fs.hpp.
  • No using directives in headers.
  • Use std:: qualifiers in headers.

Lecture 17 Review Questions

Core ideas

  • Q1. What does #include "f.hpp" do?
  • Answer: Pastes the contents of f.hpp at that location.
  • Q2. What goes in .hpp?
  • Answer: Comments and declarations.
  • Q3. What goes in .cpp?
  • Answer: Function definitions.
  • Q4. Why does functions.cpp include functions.hpp?
  • Answer: So the compiler checks the definition matches the declared signature.

Reasoning

  • Q5. Why no using namespace in headers?
  • Answer: It forces every includer to use that namespace.
  • Q6. Why are headers safe to include multiple times in a project?
  • Answer: Multiple declarations of the same function are allowed.
  • Q7. What does the linker do?
  • Answer: It connects function calls in one object file to definitions in another.
  • Q8. Why is .h versus .hpp a convention?
  • Answer: .hpp often signals "has a paired .cpp"; behavior is identical.
