Zero-Based Indexing

            
string alphabet = "abcdefghijklmnopqrstuvwxyz";

cout << alphabet[0] << endl;
cout << alphabet[1] << endl;
cout << alphabet[25] << endl;
cout << alphabet[alphabet.length() - 1] << endl;
  • Character indexing starts at 0.
  • The last character is at index length() - 1.
  • If a string has length n, its character indices run from 0 to n - 1.

[] Versus at()

[]

  • Short and common.
  • No bounds checking.
  • s[pos] is undefined behavior when pos > s.length().
  • s[s.length()] is usually a mistake, even though the textbook notes it is not undefined behavior.

at()

  • Member function call.
  • Checks bounds.
  • s.at(pos) gives a runtime error when pos >= s.length().

Mutating Characters

            
string s = "bruin";

s[0] = 'B';
s.at(1) = 'R';
s.at(2) += 'A' - 'a';

cout << s << endl;
  • Indexing is not only for reading. It can also change characters.
  • The same caution about bounds still applies.
  • Character arithmetic still works inside strings because each element is a char.

Out-of-Bounds Rules

Undefined behavior

  • s[pos] with pos >= s.length()
  • s[s.length()] = ch

Runtime error

  • s.at(pos) with pos >= s.length()
While learning, at() is often better because it fails loudly instead of silently corrupting your assumptions.

What Is size_t Doing Here?

            
#include <cstddef>
#include <string>

string s = "hello";
size_t n = s.length();

cout << n << endl;
  • length() and size() return std::size_t.
  • It is an unsigned integer type meant for sizes and indices.
  • We will keep seeing size_t throughout strings and vectors.

Lecture 5 Summary

Main ideas

  • Strings use zero-based indexing.
  • [] is unchecked; at() is checked.
  • The last valid index is length() - 1.
  • Indexing can read or modify characters.

Programming habits

  • Be suspicious whenever you see length() and an index in the same expression.
  • Use at() while learning or debugging.
  • Remember that size_t is the type of string sizes.

Lecture 5 Review Questions

Core ideas

  • Q1. If string s = "cat";, what does s[0] refer to?
  • Answer: The first character, 'c'.
  • Q2. If a string has length n, what is the last valid index?
  • Answer: n - 1
  • Q3. What is the main safety difference between [] and at()?
  • Answer: at() checks bounds, while [] does not.
  • Q4. What type do length() and size() return?
  • Answer: std::size_t

Practice

  • Q5. Why is s[s.length()] suspicious?
  • Answer: Because the last valid index is s.length() - 1, so s.length() is already out of range.
  • Q6. Can indexing change characters, or only read them?
  • Answer: It can do both.
  • Q7. Which is safer for beginners: [] or at()?
  • Answer: at()
  • Q8. Why do we call this “zero-based indexing”?
  • Answer: Because counting positions starts at 0, not 1.

Lecture 5 Code Review

Undefined behavior or not?

  • Q1. Classify the issue. 
                      
string s = "0123";
cout << s[s.length()] << endl;
  • Answer: Undefined behavior. s.length() is one past the last valid index.

Runtime error or not?

  • Q2. Classify the issue. 
                      
string s = "0123";
cout << s.at(10) << endl;
  • Answer: Runtime error, because at() checks bounds and index 10 is invalid.
