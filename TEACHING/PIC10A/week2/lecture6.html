More String Tools

Today we make strings more useful: cut pieces out with substr, grow and shrink with push_back and pop_back, and search with find and rfind.

Chapter 6.3 substr find / rfind npos

Review from Lecture 5

What we already know

  • Strings use zero-based indexing.
  • [] is unchecked and at() is checked.
  • length() returns a size_t.

What we add today

  • Extracting pieces
  • Appending and removing characters
  • Searching for text inside text

Roadmap

Cut substr(start, count) Extract a smaller string.
Search find and rfind Return positions as size_t.

substr

            
string campus = "Westwood";

cout << campus.substr(0, 4) << endl;
cout << campus.substr(4, 4) << endl;
cout << campus.substr(2, 3) << endl;
  • The first argument is the starting index.
  • The second argument is how many characters to take.
  • The result is a new string; campus itself is unchanged.

push_back and pop_back

            
string s = "Bruin";

s.push_back('s');
cout << s << endl;

s.pop_back();
cout << s << endl;
  • push_back(ch) adds one character at the end.
  • pop_back() removes the last character.
  • These are convenient when building or trimming text step by step.

find and rfind

            
string s = "bananas";

size_t p1 = s.find("ana");
size_t p2 = s.rfind("ana");

cout << p1 << endl;
cout << p2 << endl;
  • find looks from the left.
  • rfind looks from the right.
  • Both return positions, so the type is size_t.

When Search Fails: npos

            
string school = "ucla";

size_t pos = school.find("x");
cout << (pos == string::npos) << endl;
  • If the target is not found, the answer is string::npos.
  • Do not guess a fake index like -1.
  • Because the result type is size_t, comparing to string::npos is the standard pattern.

A Common find Bug

            
string name = "amy";

if (name.find("a")) {
    cout << "found" << endl;
}
  • This code is wrong when the answer is position 0.
  • 0 converts to false, so the branch is skipped.
  • Use name.find("a") != string::npos instead.

Lecture 6 Summary

Main ideas

  • substr extracts smaller strings.
  • push_back and pop_back grow and shrink strings.
  • find and rfind return positions as size_t.
  • Failed searches return string::npos.

Programming habits

  • Test searches with != string::npos.
  • Remember that search answers are indices, not booleans.
  • Read substr(start, count) carefully: start first, then how many.

Lecture 6 Review Questions

Core ideas

  • Q1. What does substr(2, 3) mean?
  • Answer: Start at index 2 and take 3 characters.
  • Q2. What does push_back add to a string?
  • Answer: One character at the end.
  • Q3. What does pop_back remove?
  • Answer: The last character.
  • Q4. What is the difference between find and rfind?
  • Answer: find searches from the left and rfind searches from the right.

Search logic

  • Q5. What does a failed search return?
  • Answer: string::npos
  • Q6. Why is if (s.find("a")) dangerous?
  • Answer: Because a match at index 0 converts to false.
  • Q7. What type do find and rfind return?
  • Answer: size_t
  • Q8. How do you test whether s.find("x") succeeded?
  • Answer: Check whether the result is not equal to string::npos.
