Classes: Structs and Member Variables

Built-in types model single numbers and characters. To model a point or a time, we bundle the related values into one user-defined type.

Chapter 15 struct member variables objects

  • Function overloading: one name, many parameter lists.
  • The compiler picks the best viable match.

Today (Ch 15.1)

The Problem: Loose Variables

            
int hour   = 9;
int minute = 30;
int second = 0;

// To pass "a time" around, every function needs three parameters:
void print_time(int h, int m, int s);
print_time(minute, hour, second);   // compiles, but wrong order!
  • Three variables that always travel together.
  • Nothing stops you from swapping the order at a call.
  • We want one value that is a time.

Declaring a struct

            
struct Time {
    int hour;
    int minute;
    int second;
};   // <-- semicolon is required
  • Time is now a type, usable like int.
  • hour, minute, second are member variables.
  • The closing brace must be followed by a semicolon.

Definition

A struct is a user-defined type that groups several member variables under one name. A variable of that type is called an object; each object has its own copy of every member.

Creating Objects

            
Time t1;        // an object; its three int members are uninitialized
Time t2;        // a second, independent object

// t1 and t2 are separate: 6 ints total, 3 per object
  • Time t1; declares an object the same way int n; does.
  • Each object bundles all three members in one variable.
  • Members of built-in type start uninitialized here.

Initializing by Assignment

            
Time t2;
t2.hour   = 14;
t2.minute = 5;
t2.second = 45;
  • object.member names one member of that object.
  • Each member is assigned like an ordinary int.
  • This works, but takes one statement per member.

Initializing with Braces

            
Time t1{9, 30, 0};      // hour=9, minute=30, second=0
Time noon{12, 0, 0};    // members filled in declared order
  • Brace init fills members in the order they were declared.
  • {9, 30, 0} → hour=9, minute=30, second=0.
  • One concise line creates a fully set-up object.

The Dot Operator

            
Time t{9, 30, 0};

std::cout << t.hour << ":" << t.minute << "\n";   // read: 9:30
t.minute = t.minute + 15;                          // write
std::cout << t.minute << "\n";                      // 45
  • t.hour reaches into one member of object t.
  • Members are read and written like normal variables.
  • Writing t.minute changes only t's copy.

Each Object Is Independent

            
Time a{1, 0, 0};
Time b{0, 1, 30};

a.hour = 5;                  // changes a only
std::cout << a.hour << "\n"; // 5
std::cout << b.hour << "\n"; // 0  (unaffected)
  • Two objects of the same type hold separate members.
  • Modifying a never touches b.
  • This is just like two separate int variables.

Passing a struct by Value

            
void show(Time t) {                      // a copy of the object
    std::cout << t.hour << ":" << t.minute
              << ":" << t.second << "\n";
}

Time now{12, 0, 0};
show(now);   // ONE argument carries the whole time
  • One parameter replaces three loose ints.
  • By value copies all members into the parameter.
  • Changes inside show would not affect now.

Passing by const Reference

            
void show(const Time& t) {     // no copy; cannot modify t
    std::cout << t.hour << ":" << t.minute
              << ":" << t.second << "\n";
}

Time now{12, 0, 0};
show(now);
  • const Time& avoids copying the whole object.
  • const guarantees show will not change now.
  • This reuses the reference rules from earlier lectures.

Data, Not Yet Behavior

So far a struct only holds data. The logic about a time — like converting to total seconds — still lives in free functions. Next lecture we move that behavior inside the type.

Lecture 19 Summary

Main ideas

  • struct bundles related member variables into one type.
  • An object is one variable holding all members.
  • Brace init fills members in declared order.
  • The dot operator reads and writes one member.

Programming habits

  • Never forget the semicolon after the closing brace.
  • Group values that always travel together.
  • Pass big objects by const& to avoid copies.

Lecture 19 Review Questions

Core ideas

  • Q1. What is a member variable?
  • Answer: A variable that belongs to each object of the struct type.
  • Q2. How do you reach one member of an object?
  • Answer: The dot operator: object.member.
  • Q3. In Time t{9, 30, 0};, what is t.second?
  • Answer: 0; brace init fills members in declared order.

Reasoning

  • Q4. If a and b are both Time, does a.hour = 5; change b?
  • Answer: No; each object has its own independent members.
  • Q5. Why pass const Time& instead of Time?
  • Answer: It avoids copying the object and guarantees it is not modified.
  • Q6. Does a plain struct contain any behavior yet?
  • Answer: No; it only holds data — member functions come next.

Lecture 19 Code Review

What prints?

  • Q1. Output? 
                
struct Point { double x; double y; };

Point p{3, 4};
Point q = p;        // copies both members
q.x = 99;
std::cout << p.x << " " << q.x << "\n";
  • Answer: 3 99 — q is a separate copy, so changing q.x leaves p.x untouched.

Spot the bug

  • Q2. Why won't this compile? 
                
struct Time {
    int hour;
    int minute;
    int second
}

Time t{9, 30, 0};
  • Answer: Two bugs: int second is missing its semicolon, and the struct definition is missing the required semicolon after its closing brace.
