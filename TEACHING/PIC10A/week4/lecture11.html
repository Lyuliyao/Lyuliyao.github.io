Functions I

Functions let us name reusable pieces of code. We start with declarations, definitions, calls, parameters, arguments, and signatures.

Chapter 9.1 declarations definitions parameters common errors

Review

From control flow

  • Branches and loops organize instructions inside one function.
  • Now we learn how to create more functions.

Why functions

  • They reduce repeated code.
  • They give names to ideas and make programs easier to test.

Roadmap

First declarations Tell the compiler which helper functions exist.
Then definitions and calls Give function bodies and use them from calling code.
Next common errors Missing declarations, missing definitions, repeated definitions.
Finally return rules void, returned values, and missing-return undefined behavior.

Function Declaration

            
int my_max(int i, int j);
void print(double d);

// Parameter names may be omitted in declarations:
int my_max(int, int);
void print(double);
  • A declaration ends with a semicolon.
  • It gives the return type, name, and parameter types.
  • It is enough for the compiler to understand later calls.

Function Definition

            
int my_max(int i, int j) {
    if (i > j) {
        return i;
    }
    return j;
}
  • A definition includes the body in braces.
  • Parameter names are needed when the body uses them.
  • Only the code path reached by the call actually executes.

Calling a Function

            
int c = my_max(a, b);
int k = my_max(j, i);
print(1.01);
  • Calling means using the function.
  • Arguments are supplied at the call site.
  • A non-void returned value can be assigned or used in a larger expression.

Build Error: Call Before Declaration

            
int main() {
    cout << triple(4) << endl;  // build error
}

int triple(int n) {
    return 3 * n;
}
  • The compiler reads the file from top to bottom.
  • At the call, it has not seen any declaration for triple.
  • Fix it by declaring triple before main, or defining it before main.

Build Error: Declared But Not Defined

            
int add(int a, int b);

int main() {
    cout << add(2, 5) << endl;
}

// No definition of add appears anywhere.
  • The compiler can understand the call from the declaration.
  • The build still fails because no body exists to run.
  • More specifically, this is a linker error.

Repeated Declarations and Definitions

Legal

            
int add(int a, int b);
int add(int a, int b);  // legal repeated declaration

int add(int a, int b) {
    return a + b;
}

Build error

            
int add(int a, int b) {
    return a + b;
}

int add(int x, int y) {  // build error
    return x + y;
}
A non-inline function may be declared multiple times, but it may be defined only once in a program.

Parameters

            
int my_max(int i, int j) {
    if (i > j) {
        return i;
    }
    return j;
}
  • i and j are parameters.
  • Parameters exist to be used inside the function definition.
  • Each function call gets its own parameter variables.

Arguments

            
int a = 3;
int b = 7;
int c = my_max(a, b);

int i = 5;
int j = 4;
int k = my_max(j, i);
  • a, b, j, and i are arguments.
  • Arguments are provided when the function is called.
  • Match arguments to parameters by position, not by name.

Name Conflicts Are Resolved by Position

            
int my_max(int i, int j);

int i = 5;
int j = 4;

int k = my_max(j, i);
// first argument j initializes parameter i
// second argument i initializes parameter j
  • The parameter names belong to the function.
  • The argument names belong to the caller.
  • The first argument goes to the first parameter, and so on.

Function Signatures

            
int my_max(int i, int j);
void print(double d);

// Same signatures:
int my_max(int, int);
void print(double);
  • A signature includes the function name.
  • It includes the number and types of parameters.
  • The textbook also lists the return type as part of the signature description here.

void Return Type

            
void print(double d) {
    cout << "printing... " << d << endl;
    // return;  // optional at the end
}

// void x;  // build error
  • void has no values.
  • A return type of void means no value is returned.
  • A void function may still use plain return; to end the call.

Undefined Behavior: Missing Return

            
int abs_val(int i) {
    if (i > 0) {
        return i;
    }
    if (i < 0) {
        return -i;
    }
}

int v = abs_val(0);  // undefined behavior
  • abs_val has return type int.
  • When i == 0, the body reaches the closing brace without returning.
  • Using the missing return value is undefined behavior.

Fix: Return on Every Path

            
int abs_val(int i) {
    if (i >= 0) {
        return i;
    }
    return -i;
}
  • For a non-void function, ask: does every possible path return?
  • Warnings about missing return are serious even when the program builds.
  • Do not rely on whatever value seems to appear during one test run.

Lecture 11 Summary

Main ideas

  • Function declarations announce functions before they are used.
  • Function definitions provide the executable body.
  • Arguments are provided by calls; parameters live inside the function call.
  • Function signatures document the return type, name, and parameter types.
  • Repeated matching declarations are legal, but repeated non-inline definitions are not.

Error habits

  • Put declarations before calls.
  • Make sure every declared function that is called has exactly one definition.
  • For non-void functions, make every path return a value.
  • Use names that describe the purpose of the function.

Lecture 11 Review Questions

Core ideas

  • Q1. What does a function declaration tell the compiler?
  • Answer: The return type, name, and parameter types.
  • Q2. What does a function definition add?
  • Answer: The function body.
  • Q3. What is a function call?
  • Answer: An instruction to run the function.
  • Q4. What is a parameter?
  • Answer: A variable listed in the function definition/declaration.
  • Q5. Is it legal to declare a function multiple times?
  • Answer: Yes, if the declarations match.

Reasoning

  • Q6. What is an argument?
  • Answer: A value supplied in a function call.
  • Q7. Can a function be declared before it is defined?
  • Answer: Yes.
  • Q8. What should you check when a call does not build?
  • Answer: Whether a matching declaration is visible before the call.
  • Q9. What if a called function was declared but never defined?
  • Answer: The build fails because there is no function body to connect to the call.
  • Q10. What if a non-void function reaches the end without returning?
  • Answer: Using the missing return value is undefined behavior.

Lecture 11 Code Review

Missing declaration

  • Q1. Why does this fail? 
                
int main() {
    cout << triple(4) << endl;
}

int triple(int n) {
    return 3 * n;
}
  • Answer: triple is called before the compiler has seen a declaration.

Missing return

  • Q2. What is wrong with this function? 
                
int f(int x) {
    if (x > 0) {
        return x + 1;
    }
}

int y = f(0);
  • Answer: It builds on some compilers, but f(0) reaches the end without returning an int, so using the result is undefined behavior.
