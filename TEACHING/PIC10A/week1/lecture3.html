Why We Need double

int was not enough

  • int stores integers only.
  • Many quantities have fractional parts: averages, distances, probabilities, slopes.
  • That is where double becomes useful.

Idea

  • double stands for double-precision floating point.
  • It can represent many decimal-looking numbers, but not every real number exactly.
  • That “not exactly” will matter later in the lecture.
Use double when the quantity naturally has a fractional part.

double Arithmetic

            
#include <iostream>
using namespace std;

int main() {
    double d1 = 1.25;
    double d2 = 0.5;

    cout << d1 + d2 << endl;
    cout << d1 - d2 << endl;
    cout << d1 * d2 << endl;
    cout << d1 / d2 << endl;
}
  • double supports +, -, *, /.
  • When both operands are double, division is ordinary fractional division.
  • Here 1.25 / 0.5 gives 2.5.
Dividing by 0.0 is undefined behavior in the course model. Do not do it.

What Is a double?

Machine model

  • A double usually uses 8 bytes, or 64 bits.
  • It is designed to store a wide range of numbers with fractional parts.
  • But it stores approximations, not exact real numbers.

Why this matters

  • double is much better than int for measurements and averages.
  • However, some decimals such as one-third cannot be represented exactly.
  • So floating-point arithmetic is useful, but it comes with precision questions.
Good mental model: a double is a practical approximation to a real number, not a perfect copy of one.

Casting Between int and double

            
#include <iostream>
using namespace std;

int main() {
    int i = 2;
    double x = static_cast<double>(i);

    double d = 4.8;
    int truncated = static_cast<int>(d);
    cout << x << " " << truncated << endl;
}
  • static_cast<double>(i) turns 2 into 2.0.
  • static_cast<int>(4.8) discards the fractional part and gives 4.
  • Prefer explicit static_cast when a conversion matters to the meaning of the code.

Mixed Division and the Average Bug

            
#include <iostream>
using namespace std;

int main() {
    int i1 = 0, i2 = 0, i3 = 1;
    int sum = i1 + i2 + i3;

    cout << sum / 3 << endl;
    cout << sum / 3.0 << endl;
    cout << static_cast<double>(sum) / 3 << endl;
}
  • sum / 3 performs int division and prints 0.
  • As soon as one side is a double, the result is a double.
  • So sum / 3.0 and static_cast<double>(sum) / 3 work for averages.

<cmath> Gives Common Math Functions

            
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    cout << sqrt(9.0) << endl;
    cout << pow(2.0, 8.0) << endl;
    cout << cos(0.0) << endl;
    cout << acos(-1.0) << endl;
}
  • Include <cmath> when you want standard math functions.
  • Useful examples: sqrt, pow, cos, acos.
  • acos(-1.0) is a common way to produce pi.

Precision of double

            
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double x = 1.0 / 3.0;
    cout << x << endl;
    cout << setprecision(2)  << x << endl;
    cout << setprecision(5)  << x << endl;
    cout << setprecision(10) << x << endl;
}
  • Many decimals cannot be represented exactly in binary floating point.
  • setprecision changes how many digits you display.
  • Changing the printout does not change the stored approximation.

Why == 0.0 Can Surprise You

            
double i = 4.0, j = 3.0;
double k = i / j - 1.0 - 1.0 / 3.0;

cout << k << endl;
cout << boolalpha << (k == 0.0) << endl;
  • Mathematically, that expression should be 0.
  • In floating-point arithmetic, tiny representation errors can remain.
  • So exact equality with doubles needs caution.
For now, the important lesson is conceptual: floating-point numbers are approximations.

unsigned int and std::size_t

unsigned int

  • It stores only non-negative integers.
  • Subtracting from 0 can wrap around to a very large value.
  • So it is not automatically “safer” than int.

std::size_t

  • std::size_t is an unsigned integer type.
  • It is commonly used for sizes and indices.
  • We will see it again when we start strings and containers.

char Stores One Character

            
char c = 'A';
int i = 65;

cout << c << endl;
cout << static_cast<int>(c) << endl;
cout << static_cast<char>(i) << endl;
  • Character literals use single quotes: 'A'.
  • Each character has a numerical code point.
  • static_cast<int>(c) reveals the code point.

char Arithmetic Uses Code Points

            
char c = 'A';
cout << c << endl;
cout << c + 1 << endl;
cout << static_cast<char>(c + 1) << endl;
  • In arithmetic expressions, a char is promoted to an integer code point.
  • So c + 1 is a number, not automatically a character.
  • If you want the next character, cast back: static_cast<char>(c + 1).

Explicit vs Implicit Casting

            
int i = 5.9;                  // implicit truncation
char ch = 65;                 // implicit conversion to 'A'
bool b1 = static_cast<bool>(0);
bool b2 = static_cast<bool>(-3.14);

cout << ch << endl;
cout << boolalpha << b1 << " " << b2 << endl;
  • Implicit casting happens automatically.
  • Explicit casting uses static_cast<T>(expr).
  • When the conversion matters for correctness or clarity, explicit is better.
0 -> false nonzero -> true

Lecture 3 Summary

Main ideas

  • double handles quantities with fractional parts.
  • Mixed division changes once a double is involved.
  • static_cast makes type conversion explicit.
  • <cmath> adds standard math functions.

Important cautions

  • Floating-point values are approximations, so exact equality can be tricky.
  • unsigned int and std::size_t are unsigned types.
  • char values also have numeric code points.
  • Implicit casts happen automatically, but explicit casts are often clearer.
We now have enough type machinery to start working with strings and indexed data next time.

Lecture 3 Review Questions

Floating-point ideas

  • Q1. What is the difference between sum / 3 and sum / 3.0?
  • Answer: sum / 3 does integer division, while sum / 3.0 does floating-point division.
  • Q2. What does static_cast<int>(4.8) evaluate to?
  • Answer: 4
  • Q3. Why can checking k == 0.0 be unreliable for a double?
  • Answer: Because a double stores an approximation, so tiny rounding errors may remain.
  • Q4. What is the key difference between the mental model of an int and a double?
  • Answer: An int stores exact whole numbers in a finite range, while a double stores an approximation to a real number.

Types and characters

  • Q5. What is static_cast<int>('A')?
  • Answer: 65 in ASCII.
  • Q6. How would you turn a lowercase letter into uppercase using character arithmetic?
  • Answer: One common pattern is static_cast<char>(ch + 'A' - 'a').
  • Q7. What is std::size_t typically used for?
  • Answer: Sizes and indices.
  • Q8. Why should you be cautious with unsigned int?
  • Answer: It can wrap around in ways that are easy to misread.

Lecture 3 Code Review

double traps

  • Q1. This is adapted from the chapter review. The code builds. What are the two subtle mistakes in reasoning? 
                      
int i = 3;
int j = 4;
int k = 20;

double d1 = (i + j) / 2;
cout << d1 << endl;

double d2 = 20.15;
cout << boolalpha << (d2 - k == 0.15) << endl;
  • Answer: d1 becomes 3, not 3.5, because the division is still integer division before the result is stored in a double. The equality test with 0.15 is unreliable because double stores approximations.

char reasoning

  • Q2. This is adapted from the chapter review. What is the output? 
                      
char ch = '2';
cout << boolalpha << (ch == 2) << endl;
cout << ('B' - 'a' == 'F' - 'e') << endl;
  • Answer: The output is false and then true. The character '2' is not the same as the integer 2, and both subtraction expressions produce the same integer value.
