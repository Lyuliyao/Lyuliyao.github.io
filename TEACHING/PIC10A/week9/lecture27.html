Pointer to Const

            
const int x = 5;
const int* p = &x;
// *p = 6;  // error
  • The pointer can read the int.
  • It cannot modify the int through p.
  • This is similar to a const reference.

Const Pointer

            
int x = 5;
int y = 9;
int* const p = &x;
*p = 6;
// p = &y;  // error
  • The pointer itself cannot point somewhere else.
  • The pointee can still be changed.
  • Read declarations carefully.

Null and Arrow

            
Point* p = nullptr;

if (p != nullptr) {
    cout << p->x << endl;
}
  • nullptr means no object.
  • Do not dereference null pointers.
  • p->x abbreviates (*p).x.

Constness Cases Side by Side

            
int x = 1;
int y = 2;

const int* read_only = &x;
// *read_only = 3;   // error
read_only = &y;      // ok

int* const fixed = &x;
*fixed = 3;          // ok
// fixed = &y;       // error

const int* const fixed_read = &x;
// *fixed_read = 4;  // error
// fixed_read = &y;  // error
  • const int*: cannot mutate the int through the pointer.
  • int* const: cannot reassign the pointer itself.
  • const int* const: neither operation is allowed.

this and ->

            
class Counter {
public:
    void bump() { ++this->value; }
    int get() const { return value; }

private:
    int value = 0;
};

Counter c;
Counter* p = &c;
p->bump();
cout << p->get() << endl;
  • Inside a member function, this points to the current object.
  • p->bump() means (*p).bump().
  • Use -> when the left side is a pointer to an object.
