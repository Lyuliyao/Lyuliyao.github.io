for Loops

The for loop is the standard tool when the loop variable changes in a regular way. Today we cover loop anatomy, traversal, and nested loops.

Chapter 8.5-8.6 counting loops string traversal nested loops

Review from Lecture 9

while recap

  • Initialization, condition, and update are the core loop ingredients.
  • Infinite loops happen when the update fails to move the condition toward false.

Why for exists

  • Many loops have a very regular counting pattern.
  • for keeps that pattern in one place.

Roadmap

Anatomy for (init; condition; update) All loop machinery appears in one header.
Patterns Count, traverse, nest These are the most common early uses.

The Basic Form

            
for (int i = 0; i < 5; ++i) {
    cout << i << endl;
}
  • Initialization runs once.
  • The condition is checked before each iteration.
  • The update runs after each iteration.

for Versus while

            
int i = 0;
while (i < 5) {
    cout << i << endl;
    ++i;
}
  • This is almost the same loop as the previous slide.
  • for is often clearer for regular counting.
  • while is often clearer for irregular or sentinel-driven loops.

Common Pattern: Summation

            
int sum = 0;

for (int i = 1; i <= 4; ++i) {
    sum += i;
}
  • A loop variable often walks through a numeric range.
  • Inside the loop, we accumulate or compute something from each value.

Traversing a String by Index

            
string s = "bruins";

for (size_t i = 0; i < s.length(); ++i) {
    cout << s[i] << endl;
}
  • This pattern connects loops to string indexing.
  • Notice the use of size_t for the index.
  • The stopping condition is still “stay strictly below the length.”

Nested Loops

            
for (int row = 1; row <= 3; ++row) {
    for (int col = 1; col <= 3; ++col) {
        cout << row * col << " ";
    }
    cout << endl;
}
  • The inner loop runs completely for each outer-loop step.
  • Nested loops are natural for tables, grids, and repeated patterns.

Lecture 10 Summary

Main ideas

  • for packages initialization, condition, and update into one loop header.
  • It is ideal for regular counting and traversal.
  • Strings are commonly traversed with indexed for loops.
  • Nested loops let us work with tables and two-dimensional patterns.

Programming habits

  • Check your loop bounds carefully.
  • Read loop headers left to right: init, condition, update.
  • Trace small examples by hand before trusting a nested loop.

Lecture 10 Review Questions

Loop anatomy

  • Q1. What are the three parts of a for loop header?
  • Answer: Initialization, condition, and update.
  • Q2. When is for usually clearer than while?
  • Answer: When the loop follows a regular counting pattern.
  • Q3. Why is i < s.length() a common string traversal condition?
  • Answer: Because the last valid index is one less than the length.
  • Q4. What do nested loops let us express naturally?
  • Answer: Grids, tables, and repeated two-level patterns.

Reasoning

  • Q5. What values of i occur in for (int i = 0; i < 3; ++i)?
  • Answer: 0, 1, and 2
  • Q6. Why is <= often dangerous in an indexing loop?
  • Answer: Because it can step one past the last valid index.
  • Q7. Why might size_t be a natural choice for a string index?
  • Answer: Because string lengths and sizes are reported as size_t.
  • Q8. What should you inspect first when a loop prints too many or too few values?
  • Answer: The loop bounds and update.

Lecture 10 Code Review

Off-by-one bug

  • Q1. What is suspicious here? 
                      
for (size_t i = 0; i <= s.length(); ++i) {
    cout << s[i] << endl;
}
  • Answer: The loop reaches i == s.length(), which is already out of bounds for indexing.

Loop body mistake

  • Q2. What does this actually print? 
                      
for (int i = 0; i < 3; ++i) {
    cout << 3 << endl;
}
  • Answer: It prints the number 3 three times, not the values of i.
