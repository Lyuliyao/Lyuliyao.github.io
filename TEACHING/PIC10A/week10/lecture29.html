The Array Name Is an Address

            
int a[4] = {10, 20, 30, 40};

int* p = a;        // a converts to &a[0]
cout << *p << endl;        // 10
cout << *(a) << endl;      // 10
cout << (a == &a[0]);     // 1 (true)
  • In most expressions, a decays to a pointer to a[0].
  • int* p = a; copies that address into p.
  • Dereferencing the array name yields the first element.

Array-to-Pointer Decay

Array-to-pointer decay: in nearly every context, an array of T automatically converts to a T* pointing at its first element. The length is left behind.

int a[] Is Really int*

            
void f(int a[]) { /* a is int* here */ }
void g(int* a)  { /* identical to f  */ }

void h(int a[100]) { /* the 100 is ignored */ }
  • A parameter written int a[] is exactly int* a.
  • A size inside the brackets, like [100], is ignored.
  • All three signatures above are the same function type.

Why You Must Pass the Size

            
void print(int a[]) {
    int n = sizeof(a) / sizeof(a[0]);   // BUG: sizeof a pointer!
    for (int i = 0; i < n; ++i) cout << a[i];
}

void print(int a[], int n) {            // correct: size given
    for (int i = 0; i < n; ++i) cout << a[i] << ' ';
}
  • Inside the function, sizeof(a) is the pointer size.
  • The sizeof trick only works on a real array.
  • The caller knows the length; pass it explicitly.

a[i] Is *(a + i)

            
int a[4] = {5, 6, 7, 8};

a[2];        // 7
*(a + 2);    // 7  — same thing
*(2 + a);    // 7  — and so is 2[a] (please never write that)
  • Subscripting is defined as a[i] == *(a + i).
  • a + i is the address of element i.
  • This identity is the whole secret of arrays.

Pointer Arithmetic Counts Elements

            
int a[4] = {5, 6, 7, 8};
int* p = a;     // points at a[0]

p + 1;          // address of a[1], not "one byte later"
*(p + 3);       // 8
++p;            // now p points at a[1]
cout << *p;     // 6
  • p + k advances by k elements, scaled by sizeof(int).
  • Incrementing a pointer moves it to the next element.
  • The compiler handles the byte math for you.

Walking an Array With a Pointer

            
int a[5] = {1, 2, 3, 4, 5};
int sum = 0;

for (int* p = a; p < a + 5; ++p) {
    sum += *p;
}
cout << sum << endl;   // 15
  • a + 5 is the "one-past-the-end" address.
  • Loop while p < a + 5 — never dereference that end.
  • This is exactly how iterators feel later in 10B.

C-strings Are char Arrays

            
char s[6] = "hello";   // {'h','e','l','l','o','\0'}

for (int i = 0; s[i] != '\0'; ++i)
    cout << s[i];        // hello

cout << sizeof(s);      // 6: 5 letters + terminator
  • A C-string is a char array ending in '\0'.
  • The terminator marks where the text stops.
  • Always reserve one extra slot for '\0'.

The Terminator Is Easy to Forget

A C-string without its '\0' has no end. Code that scans "until '\0'" will run off the array into unrelated memory — a classic source of undefined behavior. Prefer std::string when you can.

Off-by-One and Out-of-Range

            
int a[5] = {1, 2, 3, 4, 5};

a[5];                 // out of range (valid: 0..4)
for (int i = 0; i <= 5; ++i) a[i] = 0;   // writes a[5] — UB
int* p = a + 5;       // OK to form
cout << *p;           // NOT OK to dereference
  • You may compute the one-past-end address, not read it.
  • i <= N is the single most common array bug.
  • No diagnostic — the program just misbehaves.

Lecture 29 Summary

Main ideas

  • Arrays decay to a pointer to their first element.
  • A parameter int a[] is just int*.
  • a[i] is defined as *(a + i).
  • Pointer arithmetic moves by whole elements.

Programming habits

  • Always pass the size next to the array.
  • Stop loops at i < N / p < a + N.
  • Leave room for '\0'; prefer std::string.

Lecture 29 Review Questions

Core ideas

  • Q1. What does the array name a decay to?
  • Answer: A pointer to its first element, &a[0].
  • Q2. Rewrite a[3] using a pointer.
  • Answer: *(a + 3).
  • Q3. What type is the parameter in void f(int a[]);?
  • Answer: int* — the brackets are cosmetic.

Reasoning

  • Q4. Why does sizeof(a)/sizeof(a[0]) fail inside a function?
  • Answer: a is a pointer there, so sizeof(a) is the pointer size.
  • Q5. If p is int*, how far is p + 2 from p?
  • Answer: Two elements — 2 * sizeof(int) bytes.
  • Q6. What marks the end of a C-string?
  • Answer: The null terminator '\0'.

Lecture 29 Code Review

What prints?

  • Q1. Output? 
                
int a[4] = {2, 4, 6, 8};
int* p = a + 1;
cout << p[0] << ' ' << *(p + 2) << endl;
  • Answer: 4 8. p points at a[1], so p[0]==a[1]==4 and *(p+2)==a[3]==8.

Spot the bug

  • Q2. What is wrong? 
                
char s[5] = {'h','e','l','l','o'};
cout << s;   // print as a C-string
  • Answer: No '\0' terminator, so cout << s runs past the array — undefined behavior. Need char s[6] ending in '\0'.
