Why We Need double

int was not enough

  • int stores integers only.
  • Many quantities have fractional parts: averages, distances, probabilities, slopes.
  • That is where double becomes useful.

Idea

  • double stands for double-precision floating point.
  • It can represent many decimal-looking numbers, but not every real number exactly.
  • That “not exactly” will matter later in the lecture.
Use double when the quantity naturally has a fractional part.

double Arithmetic

            
#include <iostream>
using namespace std;

int main() {
    double d1 = 1.25;
    double d2 = 0.5;

    cout << d1 + d2 << endl;
    cout << d1 - d2 << endl;
    cout << d1 * d2 << endl;
    cout << d1 / d2 << endl;
}
  • double supports +, -, *, /.
  • When both operands are double, division is ordinary fractional division.
  • Here 1.25 / 0.5 gives 2.5.
Dividing by 0.0 is undefined behavior in the course model. Do not do it.

Casting Between int and double

            
#include <iostream>
using namespace std;

int main() {
    int i = 2;
    double x = static_cast<double>(i);

    double d = 4.8;
    int truncated = static_cast<int>(d);
    cout << x << " " << truncated << endl;
}
  • static_cast<double>(i) turns 2 into 2.0.
  • static_cast<int>(4.8) discards the fractional part and gives 4.
  • Prefer explicit static_cast when a conversion matters to the meaning of the code.

Mixed Division and the Average Bug

            
#include <iostream>
using namespace std;

int main() {
    int i1 = 0, i2 = 0, i3 = 1;
    int sum = i1 + i2 + i3;

    cout << sum / 3 << endl;
    cout << sum / 3.0 << endl;
    cout << static_cast<double>(sum) / 3 << endl;
}
  • sum / 3 performs int division and prints 0.
  • As soon as one side is a double, the result is a double.
  • So sum / 3.0 and static_cast<double>(sum) / 3 work for averages.

<cmath> Gives Common Math Functions

            
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    cout << sqrt(9.0) << endl;
    cout << pow(2.0, 8.0) << endl;
    cout << cos(0.0) << endl;
    cout << acos(-1.0) << endl;
}
  • Include <cmath> when you want standard math functions.
  • Useful examples: sqrt, pow, cos, acos.
  • acos(-1.0) is a common way to produce pi.

Precision of double

            
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double x = 1.0 / 3.0;
    cout << x << endl;
    cout << setprecision(2)  << x << endl;
    cout << setprecision(5)  << x << endl;
    cout << setprecision(10) << x << endl;
}
  • Many decimals cannot be represented exactly in binary floating point.
  • setprecision changes how many digits you display.
  • Changing the printout does not change the stored approximation.

Why == 0.0 Can Surprise You

            
double i = 4.0, j = 3.0;
double k = i / j - 1.0 - 1.0 / 3.0;

cout << k << endl;
cout << boolalpha << (k == 0.0) << endl;
  • Mathematically, that expression should be 0.
  • In floating-point arithmetic, tiny representation errors can remain.
  • So exact equality with doubles needs caution.
For now, the important lesson is conceptual: floating-point numbers are approximations.

unsigned int

What it means

  • unsigned int stores only non-negative integers.
  • On most machines it ranges from 0 to 2^32 - 1.
  • It is not just “an int but better.” It behaves differently.

Why we are cautious

              
unsigned int u = 0;
u = u - 1;
cout << u << endl;

Subtracting 1 from 0 wraps to a huge value.

Use unsigned only when there is a real reason. Otherwise, ordinary int is usually easier to reason about.

std::size_t

What it is

  • std::size_t is an unsigned integer type.
  • It is used for sizes and indices.
  • We will see it again immediately when we start strings.

Why mention it now?

  • Library functions like string::size() return it.
  • So the language is already preparing us for indexing collections.
  • For now, just recognize the name and the fact that it is unsigned.

char Stores One Character

            
char c = 'A';
int i = 65;

cout << c << endl;
cout << static_cast<int>(c) << endl;
cout << static_cast<char>(i) << endl;
  • Character literals use single quotes: 'A'.
  • Each character has a numerical code point.
  • static_cast<int>(c) reveals the code point.

char Arithmetic Uses Code Points

            
char c = 'A';
cout << c << endl;
cout << c + 1 << endl;
cout << static_cast<char>(c + 1) << endl;
  • In arithmetic expressions, a char is promoted to an integer code point.
  • So c + 1 is a number, not automatically a character.
  • If you want the next character, cast back: static_cast<char>(c + 1).

Explicit vs Implicit Casting

            
int i = 5.9;                  // implicit truncation
char ch = 65;                 // implicit conversion to 'A'
bool b1 = static_cast<bool>(0);
bool b2 = static_cast<bool>(-3.14);

cout << ch << endl;
cout << boolalpha << b1 << " " << b2 << endl;
  • Implicit casting happens automatically.
  • Explicit casting uses static_cast<T>(expr).
  • When the conversion matters for correctness or clarity, explicit is better.

Lecture 2 Summary

Main ideas

  • double handles quantities with fractional parts.
  • Mixed division changes once a double is involved.
  • static_cast makes type conversion explicit.
  • <cmath> adds standard math functions.

Important cautions

  • Floating-point values are approximations, so exact equality can be tricky.
  • unsigned int and std::size_t are unsigned types.
  • char values also have numeric code points.
  • Implicit casts happen automatically, but explicit casts are often clearer.
