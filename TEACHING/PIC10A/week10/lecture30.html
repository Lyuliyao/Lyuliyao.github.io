Integrated Example: Grade Book

            
class GradeBook {
public:
    void add(int score) {
        scores.push_back(score);
    }

    double average() const {
        if (scores.empty()) {
            return 0.0;
        }

        int total = 0;
        for (size_t i = 0; i < scores.size(); ++i) {
            total += scores[i];
        }
        return static_cast<double>(total) / scores.size();
    }

private:
    vector<int> scores;
};
  • The vector stores changing state.
  • The loop computes an accumulator.
  • The const member function promises not to change the object.
  • The empty case prevents division by zero.

Final Debugging Table

Error categories

  • Build: missing declaration, signature mismatch, wrong header.
  • Runtime: out-of-bounds access, null dereference, division by zero.
  • Logic: off-by-one loop, wrong accumulator update, wrong branch condition.
  • Design: unnecessary copy, missing const, public data that should be private.

What to do first

  • Reduce the input to the smallest case that still shows the bug.
  • Write down variable values after each statement.
  • Check function signatures before changing function bodies.
  • Check bounds before changing loop logic.

Design Questions

Before writing code

  • What data changes over time?
  • What function owns each task?
  • What type communicates the intent?

Before debugging

  • What should happen on a tiny input?
  • Where can scope or lifetime end?
  • Which operation may access invalid memory?

Lecture 30 Code Review

Trace it

  • Q1. What should you trace first? 
                
vector<int> readings = {4, 7, 9};
int count = 0;

for (size_t i = 0; i < readings.size(); ++i) {
    if (readings[i] > 5) {
        ++count;
    }
}
  • Answer: Trace i and count; only values greater than 5 increment the count.

Fix the design

  • Q2. What design question matters here? 
                
class Door {
public:
    void open() { is_open = true; }
    bool open_status() const { return is_open; }

private:
    bool is_open = false;
};
  • Answer: The mutator changes private state; the const observer reports it.
