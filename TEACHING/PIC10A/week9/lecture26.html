Roadmap

Core idea addresses Today centers on addresses.
Practice pointers We use examples and code review to make pointers concrete.

Address Operator

            
int x = 42;
cout << &x << endl;
  • &x is the memory address of x.
  • Addresses identify storage locations.
  • The exact printed address is not important.

Pointer Variable

            
int x = 42;
int* p = &x;

cout << p << endl;
cout << *p << endl;
  • A pointer stores an address.
  • *p dereferences the pointer.
  • Dereferencing means access the object pointed to.

Pointer vs Reference

            
int x = 1;
int& r = x;
int* p = &x;

r = 2;
*p = 3;
  • A reference is an alias.
  • A pointer is an address value.
  • Both can access the same original object.

Trace: Reassigning a Pointer

            
int x = 1;
int y = 2;
int* p = &x;

*p = 3;   // changes x
p = &y;   // p now points to y
*p = 4;   // changes y

cout << x << " " << y << endl;
  • After *p = 3, x is 3.
  • After p = &y, dereferencing p no longer touches x.
  • Final output is 3 4.

Safe Dereference Pattern

            
int* p = nullptr;

if (p != nullptr) {
    cout << *p << endl;
} else {
    cout << "no value" << endl;
}
  • nullptr says the pointer does not currently point to an object.
  • Dereference only after you know the pointer is valid.
  • This is the pointer version of checking preconditions.
