Grade Shadowing

Bug

            
int score = 93;
char grade;

if (score >= 90) {
    char grade = 'A';  // shadows the outer grade
} else if (score >= 80) {
    grade = 'B';
} else {
    grade = 'F';
}

cout << grade << endl;  // undefined behavior

Fix

            
int score = 93;
char grade;

if (score >= 90) {
    grade = 'A';
} else if (score >= 80) {
    grade = 'B';
} else {
    grade = 'F';
}

cout << grade << endl;
The bug is not the if. The bug is declaring a new grade inside one branch.

Sum One Line of Ints

            
int sum = 0;

while (true) {
    int i;
    cin >> i;
    sum += i;

    char next = cin.get();
    if (next == '\n') {
        break;
    }
}

cout << sum << endl;
  • This is not the same as a sentinel-value loop.
  • cin >> i reads the next integer and skips whitespace first.
  • cin.get() checks whether the next character was the newline.
  • The code assumes no extra spaces before ENTER.

Is n Prime?

            
const unsigned int n = 35;  // assume n >= 2
bool n_is_prime = true;

for (unsigned int d = 2; d < n; ++d) {
    if (n % d == 0) {
        cout << n << " == " << d
             << " * " << n / d << endl;
        n_is_prime = false;
        break;
    }
}

if (n_is_prime) {
    cout << n << " is prime" << endl;
} else {
    cout << n << " is not prime" << endl;
}
  • The boolean flag starts as true and changes only if a divisor is found.
  • break is correct because one divisor is enough.
  • This simple version mishandles 0 and 1.

Primes Up To N

            
const unsigned int N = 19;

for (unsigned int n = 2; n <= N; ++n) {
    bool n_is_prime = true;

    for (unsigned int d = 2; d < n; ++d) {
        if (n % d == 0) {
            n_is_prime = false;
            break;
        }
    }

    if (n_is_prime) {
        cout << n << " ";
    }
}
cout << endl;
  • The outer loop chooses the candidate n.
  • The inner loop searches for a non-trivial divisor.
  • The flag must be reset for each new candidate.

Loop Debugging Checklist

State questions

  • Where is each loop variable initialized?
  • What condition stops the loop?
  • What statement moves the state toward stopping?

Bounds questions

  • What is the first value?
  • What is the last value that should run?
  • Could the code step one past a valid index?

8.6.6: First N Primes

            
const unsigned int N = 5;
unsigned int primes_so_far = 0;
unsigned int candidate = 2;

while (primes_so_far < N) {
    bool candidate_is_prime = true;

    for (unsigned int d = 2; d < candidate; ++d) {
        if (candidate % d == 0) {
            candidate_is_prime = false;
            break;
        }
    }

    if (candidate_is_prime) {
        cout << candidate << " ";
        ++primes_so_far;
    }
    ++candidate;
}
  • Use while because we do not know the final candidate in advance.
  • primes_so_far controls when the program stops.
  • candidate must increase every iteration, prime or not.

8.6.2: Sieve Idea

            
const size_t N = 19;
string is_prime(N + 1, 't');

is_prime[0] = 'f';
is_prime[1] = 'f';

for (size_t i = 2; i <= N; ++i) {
    for (size_t j = 2; i * j <= N; ++j) {
        is_prime[i * j] = 'f';
    }
}

for (size_t p = 0; p <= N; ++p) {
    if (is_prime[p] == 't') {
        cout << p << " ";
    }
}
  • The string stores a table of truth values using 't' and 'f'.
  • The nested loops cross out composite numbers.
  • The condition i * j <= N keeps indices valid.

Lecture 10 Summary

Main ideas

  • Shadowing can leave an outer variable uninitialized.
  • Input loops can stop using the newline, not only a sentinel value.
  • Prime tests combine flags, divisibility, and break.
  • The sieve uses nested loops and a string as a temporary table.

Programming habits

  • Pick while when the stopping candidate is not known in advance.
  • Reset flags inside the correct loop.
  • Trace the exact variable used by each line.

Lecture 10 Review Questions

Scope and state

  • Q1. What is shadowing?
  • Answer: A new inner variable has the same name as an outer variable and hides it inside the inner scope.
  • Q2. Why is the grade-shadowing example undefined behavior for score 93?
  • Answer: The inner grade gets 'A', then disappears; the outer grade is still uninitialized.
  • Q3. Why does the one-line sum use cin.get()?
  • Answer: It inspects the character after the integer to detect the newline.
  • Q4. What does break do in the prime test?
  • Answer: It stops searching once one divisor proves the number is not prime.

Debugging

  • Q5. Where must n_is_prime be reset when printing primes up to N?
  • Answer: Inside the outer loop, once for each candidate n.
  • Q6. Why does first N primes use primes_so_far?
  • Answer: The program stops by count, not by a known last candidate.
  • Q7. What does is_prime[i * j] = 'f' mean in the sieve?
  • Answer: It marks a composite number as not prime.
  • Q8. What should you trace in nested loops?
  • Answer: The outer-loop value, then every inner-loop value for that outer value.

Lecture 10 Code Review

Shadowing bug

  • Q1. What is the bug? 
                
char grade;
int score = 95;

if (score >= 90) {
    char grade = 'A';
}

cout << grade << endl;
  • Answer: The branch creates a different grade; the outer grade remains uninitialized.

Missing update

  • Q2. Why might this never stop? 
                
unsigned int primes_so_far = 0;
unsigned int candidate = 2;

while (primes_so_far < 5) {
    if (candidate == 2) {
        ++primes_so_far;
    }
    // forgot: ++candidate;
}
  • Answer: candidate stays 2, so the loop keeps counting the same candidate forever.
