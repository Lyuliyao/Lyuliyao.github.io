const Instances and const Member Functions

A const object promises never to change. To honor that promise, C++ only lets it call member functions that are themselves marked const.

Chapter 15 const const member functions const-correctness

Review

Lecture 21

  • Constructors give an object a valid initial state.
  • After construction, members hold meaningful values.

Today

  • Protect objects that must not change after creation.
  • Mark reading member functions const.
  • Pass objects by const reference safely.

A const Instance

            
struct Time {
    int hour, minute;

    Time(int h, int m) { hour = h; minute = m; }
    int getHour() { return hour; }
    void setHour(int h) { hour = h; }
};

const Time noon(12, 0);   // a const instance
noon.setHour(13);         // ERROR: would modify noon
  • const Time noon may never change after construction.
  • The constructor still runs — that is initialization, not modification.
  • setHour would change it, so the compiler forbids the call.

The Surprise: Even Getters Are Rejected

            
const Time noon(12, 0);
int h = noon.getHour();   // ERROR, even though it only reads
  • getHour only reads, but the compiler does not know that.
  • From its declaration, getHour could modify the object.
  • On a const object, only functions that promise not to modify are allowed.

The Rule

A const object may call only member functions that are themselves marked const. A member function makes that promise by writing const after its parameter list.

Marking a Member Function const

            
struct Time {
    int hour, minute;

    Time(int h, int m) { hour = h; minute = m; }

    int getHour() const { return hour; }     // trailing const
    int getMinute() const { return minute; }
    void setHour(int h) { hour = h; }        // not const
};

const Time noon(12, 0);
cout << noon.getHour();   // OK now: getHour is const
  • The trailing const applies to the implicit object.
  • It promises: "this function will not modify the object."
  • const objects may call getHour and getMinute.

Where the const Goes

            
struct Time {
    int hour, minute;

    Time(int h, int m) { hour = h; minute = m; }

    int  getHour() const { return hour; }   // reader: const
    void setHour(int h)  { hour = h;     }   // writer: no const
};
  • The const sits after the parameter list, before the body.
  • It is part of the function's signature, not a return-type detail.
  • Mark every member function that only reads as const; leave writers non-const.

What a const Member Function May Not Do

            
struct Time {
    int hour, minute;

    void show() const {
        hour = 0;          // ERROR: modifies a member
        setHour(9);        // ERROR: calls a non-const member
        cout << hour << ":" << minute;   // reading is fine
    }
    void setHour(int h) { hour = h; }
};
  • It must not modify any member variable.
  • It must not call a non-const member function.
  • It may freely read members and print them.

Inside a const Member, this Is const

Within a const member function the implicit object is treated as const. Any attempt to modify a member, or to call a non-const member function, is a compile error — the promise is enforced, not just documented.

const-Correct Getters and Setters

            
struct BankAccount {
    double balance;

    BankAccount(double b) { balance = b; }

    double getBalance() const { return balance; }  // reads
    void deposit(double a) { balance += a; }        // writes
};
  • A getter that only reads should be const.
  • A setter that modifies the object must not be const.
  • This split lets read-only code use the object without risk.

Passing Objects by const Reference

            
void printAccount(const BankAccount& a) {
    cout << a.getBalance();   // OK: getBalance is const
    a.deposit(10.0);          // ERROR: deposit is not const
}

BankAccount acct(500.0);
printAccount(acct);           // no copy, cannot modify acct
  • const BankAccount& avoids a copy and forbids changes.
  • Inside, a behaves like a const object.
  • Only const member functions may be called on it.

The Classic Compile Error

            
struct BankAccount {
    double balance;

    BankAccount(double b) { balance = b; }
    double getBalance() { return balance; }   // NOT const
};

void report(const BankAccount& a) {
    cout << a.getBalance();   // error here
}
  • "passing 'const BankAccount' as 'this' argument discards qualifiers"
  • Or: no matching member function for call to getBalance.
  • a is const, but getBalance does not promise const.

The Fix: Mark the Reader const

            
struct BankAccount {
    double balance;

    BankAccount(double b) { balance = b; }
    double getBalance() const { return balance; }   // fixed
};

void report(const BankAccount& a) {
    cout << a.getBalance();   // compiles now
}
  • The cure is almost always to add the missing const.
  • Do not remove const from the parameter — that loses the protection.
  • Make every read-only member function const from the start.

Lecture 22 Summary

Main ideas

  • A const object only calls const member functions.
  • Trailing const goes in declaration and definition.
  • A const member must not modify members or call non-const members.
  • const T& parameters behave like const objects.

Programming habits

  • Mark every read-only member function const.
  • Pass big objects by const reference.
  • Fix qualifier errors by adding const, not removing it.

Lecture 22 Review Questions

Core ideas

  • Q1. Which member functions may a const object call?
  • Answer: Only those marked const.
  • Q2. Where does the keyword const go on a member function?
  • Answer: After the parameter list, in both declaration and definition.
  • Q3. May a const member function modify a member?
  • Answer: No; it may only read members and call other const members.

Reasoning

  • Q4. Why is a non-const getter rejected on a const object?
  • Answer: Its signature does not promise const, so it could modify the object.
  • Q5. What does const BankAccount& a allow you to call on a?
  • Answer: Only const member functions; a acts like a const object.
  • Q6. How do you fix "discards qualifiers" on a.getBalance()?
  • Answer: Mark getBalance as const.

Lecture 22 Code Review

Find the bug

  • Q1. Why won't this compile? 
                
struct Time {
    int hour, minute;

    Time(int h, int m) { hour = h; minute = m; }
    int getHour() const {
        hour = hour;       // ??
        return hour;
    }
};
  • Answer: A const member function may not assign to a member. hour = hour; modifies the object; remove it and just return hour;.

Make it const-correct

  • Q2. What is the error, and the fix? 
                
struct Time {
    int hour, minute;

    Time(int h, int m) { hour = h; minute = m; }
    int getMinute() { return minute; }
};

void show(const Time& t) {
    cout << t.getMinute();
}
  • Answer: t is const but getMinute is not; the call discards qualifiers. Fix: declare int getMinute() const.
